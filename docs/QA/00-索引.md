# Next-Step 用户决策 QA · 总索引

> 本目录 `docs/QA/` 集中全程 **用户拍板** 的问答留痕，便于回溯找 bug、防丢失。
> **2026-06-18 按用户要求，QA 按两类组织**：
> - **`开发/`** —— 建设性决策（愿景 / 需求 / 接线）
> - **`bugfix/`** —— 缺陷修复轮的决策
>
> **与 `../设计决策记录.md` 区分**：那里是 **lead 自主 ADR**（D-V1.1-01~10，实现级取舍、非问用户）；本目录是 **用户拍板** 的产品 / 方向决策。

---

## 如何新增（给未来窗口的写入规矩）

- **建设性决策**（需求 / 设计 / 接线 / 愿景）→ 写 `开发/`。
- **缺陷修复轮决策** → 写 `bugfix/<轮次>.md`（每轮一份，如已有 `v1.1轮.md`；下一轮 `v1.1轮2.md` 等）。
- 每条记：**背景 → 可选项（全列）→ 推荐 + 理由 → 谁拍 → 最终选择**。
- 写完**更新本索引**对应节。
- （lead 自主的实现级取舍记 `../设计决策记录.md` ADR，不写这。）
- 此规矩同步在项目 CLAUDE.md「决策 QA 记录规矩」+ 记忆 `next-step-decision-log`。
