## Q1 · 要不要写一份"高层思想"记忆？怎么写？

**用户原话：**
> 我是不是要在记忆里写一份高层思想，大概的形容我们这个next-step需要有哪些功能，目的是让每次构建都不会偏离太多，或者是设计与实际修改时有冲突，模型能及时反应而不是一直错下去，我该如何写呢？请使用提问的方式帮助我确认需求

**经 4 问确认的需求**：
- **装什么**：使命愿景 + 不可破不变量 + 核心功能支柱 + 设计哲学（全要）。
- **与 docs 边界**：只放"浓缩灵魂"，细节指向 docs（避免与 docs 重复腐烂）。
- **何时对照**：每张卡开工前 + lead 每次拍板时（主动闸门）。
- **冲突响应**：分层——触使命/红线 → 必问用户；实现级取舍 → lead 拍板记 decisions。

## Q2 · 不变量"纯文件无 DB、本地单用户"是什么意思？

**用户原话：**
> 第一③ 纯文件无 DB、本地单用户。这是什么意思，详细解释一下，我们不需要用到db数据库吗？本地单用户又是什么意思，不能多项目？

**纯文件无 DB**：所有数据存普通文件（JSON/Markdown），不用数据库。
- 例：项目→`projects.json`；Agent 档案→`.pi/agents/<id>/{agent.md,memory.md,agent.json}`；产物+版本→`.pi/artifacts/managed/<id>/{artifact.json,versions/<n>.json}`。
- 原因：沿用 pi-web 哲学；数据量小、并发低，文件够用；透明可调试（可 cat/git/手改）；零运维（不装不配不迁移）。
- **非永久禁令**：`docs/03` 原话"MVP 先 JSON 文件落盘，**出现并发/查询需求再升级 SQLite**"。即"现在够用、需要时升级"。

**本地单用户 ≠ 单项目**（关键澄清）：
- **单用户** = 一个人用 → 免登录/鉴权/多租户；**与项目数量无关**。
- **多项目** = 完全支持，且是核心功能（Iter A"项目即工作区"）。
- 类比 **VS Code**：单用户软件，但能开无数项目。
- D1 的乐观锁是防"自己开的多个会话"并发改同一产物，不是多人协作。

## Q4 · 灵魂价值是否该加"多 agent + 自定义 + 每 agent 独立窗口"？设想"同窗口 agent team"——目前是这样设计的吗？

**用户原话：**
> 灵魂价值我觉得是，细粒度可控与可回溯加多agent，自定义每个agent的功能、skill、记忆（可以先不考虑，后期再升级），并且其每一个都是单独对话窗口大大降低了同一个窗口上下文压力，我觉得以后可以升级，同一个对话窗口，让不同的agent获取其任务，去工作，完成后提交任务报告和产物，有点像agent team。我们目前是这样设计的吗？请收集信息后回答我

**调研结论（只读 Explore 实地核对代码 + docs，带 `文件:行`）**：

| 用户的理解 | 现状判定 | 证据 |
|---|---|---|
| 多 agent + 自定义每个 agent（角色/model/skills/tools/memory） | ✅ 完全是 | `lib/domain/agent-profile-store.ts`、`components/AgentManager.tsx` |
| 细粒度可控与可回溯 | ✅ 是 | 项目隔离 + 产物逐份落盘 + D1 版本/乐观锁 |
| **每个 agent 是单独对话窗口** | 🟡 **不成立** | 全 app 仅一个 `<ChatWindow>`（`AppShell.tsx:718`）、`selectedSession` 单数；多会话靠左侧列表**切换**显示一次一个；会话**不记录属于哪个 agent**；隔离单位是"会话/项目"非"agent" |
| **同一对话窗口里 agent team 取任务/干活/交报告**（设想的升级） | ❌ **目前不是** | 最接近的 Iter C 派发是"独立模态+填表单+后台静默串行+2s 轮询状态板"（`DispatchPanel.tsx`、`orchestrator.ts:85` for 串行、`dispatch-runner.ts` fire-and-forget），非对话式实时 team |

**docs 是否规划此方向**：❌ 没有。`team` 在 `docs/` 零命中；协作就 Iter C 一种；roadmap 止于 Iter D；PRD（`docs/01-prd`）把相邻的"通用意图路由 Orchestrator"列为 **Out（暂缓）**。

**结论**：当前已命中"多 agent / 自定义 / 细粒度可控可回溯"，并有"派任务→agent 干→交产物"的**雏形**（Iter C），但**还不是**"同窗口实时 agent team"。真做 = 新开一个 Iter（改会话模型、worker 前台化、对话式编排、可选并行）并重评 PRD 的 Out 决定，属产品级决策。

3. 不变量措辞改清楚：**纯文件**（可升级 SQLite）、**单用户**（≠多项目）。
