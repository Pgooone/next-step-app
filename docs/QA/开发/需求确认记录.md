## 第 3 轮（2026-06-16）— 功能#5

功能#5 拆成 5 个子需求。结构性决策（第 3 轮 a）：

- **项目首页**：选「独立的项目首页」。打开应用先看项目列表/卡片墙（增删项目，参考 pi-web 风格），点项目才进工作台。
- **@agent 含义（最关键）**：选「转交到他的单独会话里」。@agent = 把选中的上下文 + 文件作为一条消息发到该 agent 的单独会话，让他在自己会话接着干。**这是一个新的轻量功能，不等于 Dispatch，也不是就地回答。**
- **Dispatch 去留**：选「现在说不准，先保留不动」。不砍 Dispatch；@agent 与 Dispatch 并存，等 @agent 用一段时间再决定是否下线。
- **每个 agent 一个会话**：选「建档案后按需起会话，左侧标明归属」。不在建 agent 时自动冒空会话；在左侧把会话显性标成属于哪个 agent（分组或标签，细节见第 3 轮 b）。

功能#5 细节决策（第 3 轮 b）：

- **主对话 / 落地页**：选「落地到聊天工作台；主对话 = 每项目固定一个总指挥会话」。进项目直接到对话工作台，含一个固定「主对话」总入口；没设 agent 时默认用它。（不做独立项目概览页作落地页）
- **@agent 传递内容**：选「当前主对话全部历史 + 所附文件 + 发送时现勾」。默认携带全部主对话历史 + 用户所附文件，发送时允许现场勾选增减。（不限于手动选中片段）
- **左侧区分**：选「两者都要」。左栏既按「主对话 / 各 Agent」分组，每条会话又挂 agent 标签。
- **功能#4 文件大小上限**：选「不设上限」。实现侧建议加「大文件耗 token」软提示（非硬限制）。
