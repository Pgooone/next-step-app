## P0 · 架构级

### §B · 产物按块确认闭环 ⭐【V2 已闭合 —— 限 profile 会话】

> 这是首版审查的头号条目「拦截线运行时断开 = v2 命门」。**V2「提议工具」模型已把它闭合**（换架构、非接 guard）。

**现状（lead V2 实现 + 核实）**：「文档型 Agent 改产物 → 进按块确认」闭环**在 profile 会话已运行时成立**：
- 起 profile 会话（`app/api/projects/[id]/agents/[agentId]/session/route.ts:31` → `startProfileSession`）**无条件**装受限工具集（`profile-session-wiring.ts:127-142` spread `assembleDocSessionOptions`，docOptions 覆盖 profile.tools，注释 D-V2-04）。
- 受限集（`doc-session.ts:32-41` 共 7 项：read/grep/find/ls + create_artifact/propose_edit/list_artifacts）**无 write/edit/bash** → AI 改文档只能调 `propose_edit`（`doc-tools.ts:185` 落 PendingChange、不写盘）→ 按块确认 → resolve 物化出新版（`pending-change-service.ts:411`）。
- **真实数据源、非 fixture-only**：生产 propose_edit 与 resolve 路由指向同一批 `~/.pi` 受管文件，PendingChangeCard 读得到。

**首版 §B 断言已失效**：`assembleArtifactGuardOptions`（守卫）那套已**整体删除**（V2-5，`lib/pi/README.md:56-59` 标"整体移除"），不再"零调用者"——架构换成了提议工具。原 file:line `artifact-guard.ts:149` 已不存在。

**⚠️ 残留 gap（V2 只接 profile 会话）**：dispatch（见 §A）、主对话 `/api/agent/new`、idle 重建均**未装**提议工具集（grep 对 dispatch-runner / 主对话路径零引用）→ 这些入口的 AI 写产物仍不进按块确认。**北极星红线①「artifact 改动必经确认」目前仅在 profile 会话被强制**；把闭环扩到 dispatch / 主对话是下一轮承重墙。

### §A · Dispatch 派发产物不进受管系统 🟦【有意分期 · 仍是 gap】

**现状（核实，V2+bug-fix 均未触及）**：Dispatch 用 `lib/domain/orchestrator.ts:207-221` 私有 `writeArtifact`（裸 `writeFileSync`）落 `.pi/artifacts/<dispatchId>/<seq>-<agentName>.md`，与受管 `.pi/artifacts/managed/<id>/` **物理隔离**、不经 ArtifactService、不生成 PendingChange/版本；`ArtifactPicker`（数据源 = `listArtifacts` managed-only）列不到 → 到不了按块确认/版本。

**性质**：🟦 有意分期（`orchestrator.ts` 头注释 D-C-1「轻量普通文件，不版本化/不 Diff，留 Iter D」是当年刻意决策）。V2 提议工具只接 profile 会话、对 dispatch 零引用，故 V2 收官未覆盖此条、现状与首版一致。**补法**：dispatch 产物改走 `ArtifactService.createArtifact` 受管落盘（属新功能）。
