## 三、模块独立性（为什么这样划）

每个模块都过了三连问：能单独理解吗？能单独测吗？改它内部会破坏别人吗？

- **M1 / M2 / M3** 各自只碰一个组件、无共享状态 → 天然独立，可分别单测。
- **M4** 只重写 AgentManager 的**视图层**，底层 `useAgentStore`（create/update/remove/startSession）不动 → 数据层与其它模块隔离。
- **M5 是承重墙**：它把「会话归属」这件全局的事**收敛到一个领域模块 + 一个 store + 一组 API**，让 M7/M8 只依赖这个映射接口、而不互相纠缠。M5 接口定死，M7/M8 才能各自独立实现、独立测。
- **M6** 只加一层「项目墙 → 工作台」导航，复用现成项目 API，不碰会话 / agent → 独立。

## 四、依赖图与开发批次

```
批次 1（无依赖，可并行）：  M1   M2   M3   M5(地基)   M6
                                       │
批次 2（依赖 M5）：         M7 ◀────────┘        M4（独立·UI 重活，排这批）
                            │
批次 3（依赖 M5 + M7）：    M8 ◀────────────────┘
```

- **批次 1**：5 个互相独立的模块 —— 4 个快赢（M1/M2/M3/M6）+ 1 个地基（M5）。M5 虽无依赖，但要先做好，好让批次 2 接它。
- **批次 2**：M7（主对话 / 会话归属 / 左侧，接 M5）+ M4（玻璃感重做，独立但 UI 重，单独成批，避免和别的模块抢 `AgentManager`）。
- **批次 3**：M8（@agent 转交，接 M5 + M7）。
