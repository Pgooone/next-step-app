# Next-Step V1.1 文档评审报告（agent team review）

> 来源：4 个 opus 评审员（team `v1-1-docs-review`）对抗式评审，2026-06-17。
> 视角：一致性&完整性 / 代码现实性 / 北极星合规 / 可执行性&清晰度。
> 体例：每条 `[严重度·视角] 发现` → **lead 处置**。

## 总评

**方向无误、地基扎实**：7 需求 → 8 模块 → 任务全链路可追、无遗漏；5 条不可破不变量全守；3 条 lead 决策（D-V1.1-01~03）经对抗核验站得住、无更优解被漏判；边界干净（功能#5 是跨会话异步转交，**未越界**北极星「同窗口实时 agent team」愿景）；所有 file:line 引用经真实源码核对**属实**。
**要改的集中在 M8（@agent 转交）把机制说死 + 若干任务卡措辞/状态对齐**，无方向性返工。

## 阻塞级

**R1 [阻塞·一致性] 决策状态自相矛盾** —— 详细设计/进度/决策记录已写「已拍板」，但 M5/M6/M8 任务卡仍写「待用户确认决策 N」。
→ **改**：3 张卡统一为「已定 D-V1.1-0N（见 docs/设计决策记录.md）」。

**R2 [阻塞·代码现实性+可执行性] M8 投递机制未点名真实链路** —— 「投递到 agent 单独会话」过抽象。真实只有两条路：① 新会话 `POST /api/projects/[id]/agents/[agentId]/session`（firstMessage=载荷，V1 已验收、不碰 .jsonl）；② 已存在会话 `sendAgentCommand(sid, prompt)`（但要求会话 live 注册，落盘未 live 的会话无 re-attach 链路）。
→ **改 + 记 D-V1.1-04**：V1.1 取「**转交 = 起新会话**」（POST /session、载荷作 firstMessage、复用 `handleAgentSessionStarted` 切会话 + 接 SSE），一举解决投递路径 + 转交后 SSE 接流。复用既有会话（②）列为**后续增强**（需先补按 sid 的 re-attach 端点）。北极星评审已确认此路不碰内核、守不变量①。

**R3 [阻塞·可执行性] `<file>` 标签格式与内核不符** —— 文档写 `<file name="文件名">`，内核 `file-processor.js:71` 实为 `<file name="<绝对路径>">`；浏览器上传无真实绝对路径，「与 @file 一致」措辞误导。
→ **改**：M2/M8/决策记录 明确「浏览器上传以**文件名**作 name（无绝对路径），与 @file 的绝对路径语义不同、本功能可接受」，删除「与 @file 一致」。
