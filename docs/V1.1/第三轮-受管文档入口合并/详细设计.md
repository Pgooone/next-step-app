## 一、目标（一句话）

把左栏文件树切成「**顶部受管文档分组**（点开进 ArtifactPanel）+ **下方普通文件树**（点开进 FileViewer）」，
删左下 `Artifacts` 按钮 + `ArtifactPicker` 弹窗；**受管机制层（侧车 versions/pending/元数据、版本/块级 Diff/按块确认/提议工具）一行不动**——本轮只动入口/展示层。

## 二、两条数据流链路

### 链路 A · 点受管文档 → ArtifactPanel
1. **数据源**：左栏受管分组首屏 `fetch GET /api/projects/[id]/artifacts` → `listArtifacts(projectId)` 扫 `.pi/artifacts/managed/<id>/artifact.json`，返回 `Artifact[]`（含 `id/title/kind/currentVersion/filePath?`，不含 content；按 title 升序。artifact-service.ts:194-210）。每条天然带 `id`。
2. **渲染**：分组列出每条（`title` + 右侧 `kind·v{currentVersion}`，复刻 ArtifactPicker.tsx:101-127 行样式、容器从模态 button 改树行）。
3. **点击**：`onClick → onOpenArtifact(a.id)` 上抛 AppShell → `handleOpenArtifact(id) = useArtifactStore.getState().open(id) + setRightPanelOpen(true)`（= 现 `handlePickArtifact` 去掉 `setArtifactPickerOpen(false)` 的纯净版）。
4. **store.open(id)**（useArtifactStore.ts:115-144）：set `selectedArtifactId=id`；并行 fetch `/api/artifacts/[id]`（读侧车 `versions/<currentVersion>.json` 的 content，**非项目根 .md**）+ `/api/artifacts/[id]/pending`（空则 `[]`）；set artifact + pendingChanges。
5. **右栏渲染**：`selectedArtifactId ? <ArtifactPanel/> : ...`（AppShell.tsx:960）互斥切 ArtifactPanel（store 驱动、无 props、**无 SSE**）。

### 链路 B · 点普通文件 → FileViewer
1. **数据源**：FileExplorer `fetchEntries(cwd)` → `GET /api/files/<path>?type=list`（route.ts:485 经 IGNORED_NAMES 过滤）。本轮 IGNORED_NAMES **新增 `.pi`**，侧车整棵内部树从 list 消失；物化的 crd.md（项目根裸文件名，与 `.pi` 判定维度不相交）仍作根级 `.md` 显示。
2. **点击**：非目录 → `onOpenFile(fullPath,name)`（FileExplorer.tsx:104）→ SessionSidebar 透传 → `handleOpenFile`（建/激活 file tab + setRightPanelOpen(true)；**该回调 DispatchPanel 也复用，绝不可删**）。
3. **右栏渲染**：`selectedArtifactId` 为空 → `activeFileTab?.filePath ? <FileViewer/>`（AppShell.tsx:962），FileViewer 对该 filePath 开 `EventSource(?type=watch)` live 刷新（FileViewer.tsx:754）。

### 互斥关系 + SSE 冲突消解
右栏完全由 `selectedArtifactId` 二选一驱动（AppShell.tsx:928/960）：受管走 ArtifactPanel（读侧车、无 SSE）、普通走 FileViewer（读磁盘、有 SSE），**永不在同一视图叠加** → 「SSE 自动刷新 ↔ EXTERNAL_MODIFIED 拒绝覆盖」语义冲突在方案二下**降级**（核心消解、残留一处写时缺口，见 D-V3-06）。

> **命门（最高优先级风险）**：受管 .md 只要还能从下方普通树用 FileViewer 打开，SSE 冲突就复活、且绕过全部受管能力。**「冲突消解」成立的唯一前提 = 去重把项目根同名 .md 从普通树剔除**（见 D-V3-04）。

## 四、cwd ≠ projectRoot 语义（对抗 major② 定稿）

**核实**：`handleCwdChange`（AppShell.tsx:175-195）切 cwd 时**不切 currentProject**（不碰 useProjectStore）；cwd 下拉列 `getRecentCwds`（跨全局所有会话，SessionSidebar:365）。故用户可在项目 A 里把 explorer cwd 切到项目 B 的会话目录，`currentProjectId` 仍是 A。`cwd≠projectRoot` 真实可能。

**定稿结论**：受管分组 = 项目级（跟 `currentProjectId`，始终显示当前项目受管文档）；普通树 = explorer 级（跟 `selectedCwd`）；去重 = 绝对路径。三者**自洽，不限制 cwd 切换**：
- `cwd==projectRoot` 时，项目根 crd.md 的 `fullPath` = artifact 绝对路径 → 被剔出普通树（避免双现）✓
- `cwd≠projectRoot` 时，artifact 绝对路径与别处目录树节点不匹配 → 不误剔（crd.md 本就不在该树）✓
- 这是当前 pi-web「会话列表按 cwd 过滤、项目独立」语义的自然延续（SessionSidebar:366-367 会话也按 selectedCwd 过滤、与 currentProjectId 独立）

## 七、验收清单

**真浏览器·必走**：
1. 含受管 artifact 的项目，左栏 Explorer 顶部出现「受管文档」分组，列出全部 `listArtifacts` 条目（title + kind·v），数量与 API 返回一致。
2. 点分组一条 → 右栏切 ArtifactPanel（title/版本下拉/按块确认可用），`selectedArtifactId` 被 set；不经 filePath 反查即成功。
3. 物化到项目根的 crd.md **只出现在受管分组、不在普通树重复**；多次刷新无闪现残留（验 `artifacts!==null` gate）。
4. 左栏底部不再有「Artifacts」按钮；无任何入口能弹出 ArtifactPicker。
5. 文件树根级看不到 `.pi`（无该行、不可展开出裸 JSON）；同树里 crd.md 仍正常显示可点 → 证 `.pi` 隐藏未误伤项目根 md。
6. 派发面板点开某产物（路径在 `.pi/artifacts/<dispatchId>/`）仍能在 FileViewer 正常打开 → 证 type=read 不受影响。
7. 受管分组可折叠；无受管或 projectId 为空时整组不显示、不报错、不发多余请求、普通树照常。
8. 点普通文件 → 右栏走 FileViewer；受管 .md 不能从普通树再用 FileViewer 打开（不在普通树出现）→ 验双入口已堵、SSE 冲突无复活路径。
9. 纯浏览受管文档（无 pending）：右栏 ArtifactPanel 正常、无「N 处待确认」，中栏无 pending-change-card；预置 pending fixture 再点开 → 中栏出现确认卡、resolve 全块后消失。

**逻辑层·单测**：`managedAbsPaths` 过滤后根级不含受管绝对路径节点；旧 artifact（filePath=undefined）经 `.filter(Boolean)` 不污染；`cwd≠projectRoot` 时不误剔同名节点（绝对路径不等）。

**门禁**：`npm run lint && npm run test` 通过；构建不因删 ArtifactPicker / orphan state 报错。

**受管机制层零回归·必验**：复跑 `artifact-service.test.ts / pending-change-service.test.ts / doc-tools.test.ts` 全绿（侧车与三个写方法未改）。

## 八、已知 gap（登记，本轮不修）

1. 物化文件被外部改名/删除后去重失配（走 id 仍可开、降级安全）。
2. 裸名碰撞已用绝对路径 key 消除；但**外部后建一个绝对路径恰等于某 artifact 物化路径的非受管 .md** 极边界仍会被剔——概率近零、登记。
3. 右栏 rollback/submitVersion 改 title/版本号后，左栏分组版本号可能过期（不经 agent_end 不 bump explorerRefreshKey）——minor，实现时若顺手可在写成功后 bump 一次 explorer 刷新。
4. 两 useEffect（artifacts fetch + fetchEntries）刷新中途瞬时不一致窗口——`artifacts!==null` gate 挡首帧、刷新中途 minor。
5. 外部改 .md 自动生成新侧车版本（MEMORY/V2 已记本期不做）；SSE 方案 B 漂移黄条（D-V3-06 移出本轮）。
