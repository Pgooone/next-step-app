## 三、留给新窗口的修复（按优先级；全量在 `docs/第一轮-基础迭代/流程漏洞审查.md`）

### P0 · 承重墙（BUG-00）—— v2 命门【最重要 / 大活 / 必须先定范围防滑坡】
**产物「按块确认」闭环运行时是断的**（lead 已亲验）：
- 拦截器 `assembleArtifactGuardOptions`（`lib/pi/artifact-guard.ts:149`）**全仓库零运行时调用者**——起会话链路从没装它，真实 Agent 用普通 write 直接写盘、不被拦成 pending。
- dispatch 产物走非受管落盘（`lib/domain/orchestrator.ts`），不进 PendingChange 系统。
- 后果：北极星红线①「artifact 改动必经按块确认」**生产未强制**；HITL UI（D3/D4/D5）只能显示手动/测试造的 pending。
- **接线点**：`lib/pi/profile-session-wiring.ts:105` 的 `createAgentSession({ ...options })` 处并入 guard 的 options。
- **滑坡（为何要先定范围）**：起会话有三条路——① profile 会话（可接）② 主对话 `/api/agent/new`（pi-web 的，红线「不碰」）③ dispatch orchestrator；且 idle 重建走 SSE `startRpcSession` 又无注入。接哪几条、idle 是否补注入，须先和用户对齐。
