# Next-Step 新手引导

> 面向**第一次使用** Next-Step 的人。读完能跑通一条完整工作流，并知道每个功能在哪、怎么用、有哪些当前限制。
> **2026-06-19 更新**：补上 V2「文档型 Agent + 提议工具」工作流（§三.8 + §四重写）；§五「当前限制」据 bug-fix 轮（toast / 进项目 cwd / 二次确认 / 文案汉化等已修）重核。流程断点见同目录 [`流程漏洞审查.md`](./流程漏洞审查.md)。

---

## 一、Next-Step 是什么

一个**多 Agent 软件工厂**本地工具（基于开源 pi-web 改造）：

- **项目即工作区** —— 每个项目独立，会话/Agent/产物互不串。
- **可自定义多 Agent** —— 为项目建专属 Agent（配模型/技能/角色/思考强度）。
- **可派发多 Agent 协作** —— 设个目标，让多个 Agent 串行接力产出文档。
- **文档型产物：提议改动 / 块级 Diff / 按块确认 / 版本** —— 让文档型 Agent 用「提议工具」建/改文档，每处改动逐块审核、留版本、物化成项目里真实 `.md`（类 Notion/Word 修订）。

特点：**本地单用户、纯文件落盘（无数据库/无登录）、并发会话 ≤ 3**。

## 二、5 步跑通主流程

```
①打开应用                      ②建/选项目              ③主对话开聊
  项目卡片墙        ── 点项目 ──▶  落到「主对话」  ── 输入 ──▶  Agent 回答
  （ProjectHome）                  （工作台 AppShell）
       │                                                          │
       │                          ④建文档型 Agent / @转交           ▼
       └─────────────────────────  让 Agent 建/改受管文档     ⑤产物按块确认
                                    （create_artifact/propose_edit）右看 Diff·左逐块 ✓/✗·留版本·物化真实 .md
```

> 最短路径：**建项目 → 进工作台 → 主对话里直接聊**就能用。文档型 Agent / 转交 / 产物确认是进阶。

## 三、功能逐个上手

### 1. 项目墙与项目（M6）
- **打开应用**先看到**项目卡片墙**，每张卡显「项目名 + 路径 + 创建于」。
- **建项目**：填名称 + 本地路径 → 入墙。**进工作台**：点项目卡。**回项目墙**：工作台内点「回到项目墙」。
- **删项目**：二次确认——**只移除注册、不删磁盘上的 `.pi/` 数据**；重新加同一路径即恢复（本地单用户下是特性）。

> 小知识：首页与工作台是**同一个页面按「当前项目」二选一渲染**（无独立 URL，故没有项目深链）。

### 2. 主对话与左栏分组（M7）
- 进项目默认落到该项目唯一的**「主对话」**。主对话不预建空会话——**你发出第一条消息时它才被自动认定为 main**。
- 左侧会话栏按归属分**三区**：**主对话** / **各 Agent 分组**（带名+色点+计数，可折叠）/ **其它会话**。切项目不串显。

### 3. 建 Agent（M4 玻璃卡片）
- 顶栏 **Agents** 按钮（**需先选项目**）→ AgentManager（玻璃质感正方形卡片网格）。
- **点卡片** → 就地弹**浮层菜单**三段：**配置**（模型/技能/角色/思考强度，现场改即时保存）/ **起会话**（输入开场白 → 起该 Agent 一条会话）/ **删除**（二次确认）。**新建**：网格点「**+**」空卡 → 填名称（必填）+ 配置。

> 字段速记：**角色描述**=职责/风格；**思考强度**=是否启用深层思考；**模型**留空=全局默认。
> ⚠️ **「工具」字段当前不生效**（V2 变化）：V2 后**任何 Agent 起会话都自动装「文档型受限工具集」**（只读探索 + 提议工具，**无 write/edit/bash**），会覆盖这里配的工具。详见 §三.8 / §四。区分「文档型 vs 编码型 Agent」是后续增强。

### 4. 主对话 @agent 转交（M8）
- 在**主对话**输入框打 **`@`**（行首或空格后）→ 弹该项目 **Agent 待选列表**（输入框已有提示「输入消息，或用 @ 转交给 Agent」）。
- 选 Agent → 弹「**转交内容勾选**」：默认勾【全主对话历史】+【已附文件】，可现场增减。
- 点「转交」→ 打包成 `<context source="主对话">…</context>`（保留 角色标注）+ `<file>` 附件，**起该 Agent 一条新会话**投递、自动切过去。

> 注意：**`@` 只在「主对话」生效**；每次转交起的是**新会话**（续接旧会话是后续增强）。

### 5. 上传文本文件（M2）
- ChatInput 左下**文件按钮**（图标是相册样，但 title 写明「附加图片或文本文件」）→ 选文件。
- 文本白名单：`.md .txt .json .csv .log .py .ts .tsx .js .jsx .html .css .yaml .yml .xml .sql .sh`。发送时内联进消息。单文件 >256KB 有软提示。
- **PDF / Word / Excel 暂不支持**（选了会提示转文本）。

### 6. 多 Agent 派发（Dispatch，V1）
- 设目标 → 派发面板让多个 Agent**串行接力**，每个产出落 `.pi/artifacts/<dispatchId>/` 下的 `.md`。
- 与 `@agent` 转交是**两套并存**：派发=自动接力；@转交=手动甩上下文给单个 Agent。
- ⚠️ 派发产物是**轻量普通 `.md`**、**不进受管系统**（到不了按块确认/版本，见 §五 + `流程漏洞审查.md` §A）。

### 7. 产物面板：Diff / 按块确认 / 版本（Iter D + M3）
- 顶栏 **Artifacts** 按钮 → 选一个受管产物 → 右侧 **ArtifactPanel** 打开。
- 有改动时：正文**行内高亮**（绿=新增/红=删除/黄=修改），右上「N 处待确认」。「→ 在左侧对话框逐块确认」「按 `D` 看并排 Diff」。
- **按块确认**：左侧 **PendingChangeCard** 逐块 **✓/✗**（或 Y/N/R/D）；**「全部 ✓ / 全部 ✗」有二次确认**；全部处理完才**物化写盘 + 生成新版本**。
- **版本管理**：版本下拉切历史版（只读）；可**回滚**到某版（**二次确认**、生成新版）。

> 红线：**受管产物的任何改动都必须经「按块确认 → 物化写盘 → 新版本」**，不会被 Agent 直接覆盖。

### 8. ⭐ 文档型 Agent：提议工具产文档（V2 新增，核心工作流）

V2 让「Agent 改受管文档 → 进按块确认」这条闭环**真正在运行时跑通了**（之前只有 UI、缺真实数据源）。**怎么用**：

1. **建/选一个 Agent**（§三.3），给它写个角色（如「资深需求分析师」）。
2. **起它的会话**（卡片菜单「起会话」），开场白直接说需求，例如：
   > "帮我新建一个『用户登录需求』文档，包含账号密码、短信验证码、第三方登录三个功能点。"
3. **Agent 自己调 `create_artifact`** 建文档 → 直接落第一版 + **物化成项目根的真实 `.md` 文件**（项目目录里直接能看到、file panel 可见，类 Notion 页）。
4. **再让它改**，例如："把第 2 点改成语音验证码登录，其它不变。"
   - Agent 调 **`propose_edit`**：**不直接写盘**，而是把改动转成**待确认变更（PendingChange）**。
   - 它**内部回的是整篇新全文**，系统用 LCS 只切出**真正变化的块**给你确认（所以"只改一段"你只会看到那一块、不会满屏噪声）。
5. **你按块确认**（§三.7）→ 确认后才物化写盘、生成新版本、同步真实 `.md`。

> **三个提议工具**（文档会话自带、Agent 自动会用）：`create_artifact`（建文档+物化）/ `propose_edit`（提议整篇修改→待确认）/ `list_artifacts`（列文档，Agent 按标题找 id 再改）。
>
> **为什么 Agent「只能提议、不能直接写」**：文档会话装的是**受限工具集**——只有只读探索（read/grep/find/ls）+ 这 3 个提议工具，**没有 write/edit/bash**。所以 Agent 结构性地**没有**直接写盘能力，改受管文档**只能**走「提议 → 你确认 → 才落盘」。审核权牢牢在你手里。
>
> **当前范围**：这条闭环目前在 **Agent 单独会话（profile 会话）** 成立。**主对话**和**派发（Dispatch）**还没接这套（见 §五 + `流程漏洞审查.md` §A/§B 残留）。

## 四、原理：高亮 / Diff / 确认怎么实现？Agent 怎么用上？

> 先讲结论：**高亮/切块/确认/版本是 Next-Step 写死的应用功能**；而 **Agent 通过「提议工具」显式地把改动送进这套机制**（V2 模型）。这跟旧版引导说的"透明拦截 write"已经不同——下面讲清现在的真实机制。

### 1. 高亮 / 切块 / 确认 / 版本 —— 全是写死的应用代码（不变）

和具体 Agent 无关，是固定能力：

| 功能 | 由谁实现 | 在哪 |
|---|---|---|
| 把「旧内容 → 新内容」切成 add/del/mod 块 | LCS 切块算法 | `lib/domain/pending-change-service.ts` |
| 绿/红/黄 行内高亮、并排 Diff | 纯渲染函数 + 面板 | `lib/artifact-view/` + `components/ArtifactPanel.tsx` |
| 逐块 ✓/✗ 确认、物化写盘 | 卡片 + resolve API + 存储 | `components/PendingChangeCard.tsx` + `pending-change-service.ts` |
| 版本提交 / 回滚 / **物化真实 .md** | 领域服务 | `lib/domain/artifact-service.ts` |

### 2. Agent 怎么「产生」一处待确认改动？—— 显式调「提议工具」（V2 模型）

> **这是 V2 对旧版引导的关键修正**。旧版说"系统给 Agent 特制的 write/edit、Agent 无感知地写、被透明拦截"——那套 **guard 方案已整体删除**。现在是**提议工具**模型：

- 起 Agent 会话时，系统装一套**受限工具集**（`lib/pi/doc-session.ts`：只读探索 + `create_artifact`/`propose_edit`/`list_artifacts`，**无 write/edit/bash**）。
- Agent 改受管文档时**显式调 `propose_edit`**（不是伪装的 write）：工具内部读当前版当 old、用 Agent 回的整篇新全文算 Diff 块、存成一条「待确认改动」（`lib/pi/doc-tools.ts`），**不写盘**。
- 于是：**Agent 调一次提议工具，你收到的是「待你逐块确认的改动」**。审核权留人手里。

> 和旧"透明拦截"相比哪点变了：**从"骗 Agent 以为在正常写"改成"给 Agent 一个明确的提议工具让它显式调"**。好处是结构更干净（受限集**没有**写盘工具 → 无绕过路径），且实测真模型（如 DeepSeek）能理解工具说明、自己选对工具、并按要求回整篇。代价是 Agent **需要**会用这几个工具（但工具说明里写清了，模型能照做）。

### 3. ✅ 现状：这套机制已接到真实会话（V2 已闭合，限 profile 会话）

旧版引导这里曾是「⚠️ 还没接线、v2 命门」。**V2 已经把这根线接上了**：

- 起 **Agent 单独会话**（`POST /api/projects/[id]/agents/[agentId]/session` → `startProfileSession`）时，**无条件**装上受限工具集（`profile-session-wiring.ts`）。所以现在 Agent 改受管文档**真的会**变成待确认改动、进按块确认。
- 按块确认 → resolve → 物化出新版本 + 同步真实 `.md`，全链跑通（已用真模型 DeepSeek 端到端验证）。

**⚠️ 仍有边界（诚实说明）**：V2 只接了 **Agent 单独会话**。**主对话**（`/api/agent/new`）和**派发（Dispatch）**还没装这套——主对话的 Agent 仍是通用会话；派发产物落普通 `.md`、不进受管。把闭环扩到这两处是下一轮的事（见 `流程漏洞审查.md` §A/§B 残留）。

## 五、新手须知（当前限制，2026-06-19 重核）

| 主题 | 现状 | 说明 |

| **操作回执** | ✅ 多数已补 **toast**（转交/起会话/确认/回滚/增删项目与 Agent 成功都有提示） | bug-fix 轮已加（少数被动诊断如"模型回退"仍只进控制台、防刷屏） |
| **`@` 转交** | ✅ 只在主对话生效，**输入框已有提示**「…或用 @ 转交给 Agent」 | 主对话输入框打 `@` |
| **进项目可用性** | ✅ 经项目墙进**全新项目**也能正常起会话（cwd 自动取项目根） | bug-fix 轮已修（旧版此处是新手第一道坎） |
| **高危操作** | ✅ 「全部 ✓/✗」、版本「回滚」**已加二次确认** | bug-fix + D5；逐块单块仍直接执行（低危、可单块回退） |
| **外部改 .md** | 受管文档物化成真实 `.md` 后，**外部（编辑器/git）直接改它不自动进版本**；下次 Agent 确认前会检测到外部改动并**拒绝覆盖**（防数据丢失） | V2 EXTERNAL_MODIFIED 保护（D-V2-06）；"外部改动自动快照"是后续 |
| **文件类型** | 仅文本类；PDF/Word/Excel 不支持 | 转文本再传 |
| **首条回答偶发缺字** | 极少数情况首条回答开头可能缺几个字（流式订阅时序） | 已知机制 gap（`流程漏洞审查.md` §H），刷新会话可见完整落盘内容 |
