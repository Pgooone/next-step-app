# Next-Step 新手引导

> 面向**第一次使用** Next-Step 的人。读完能跑通一条完整工作流，并知道每个功能在哪、怎么用、有哪些当前限制。
> 配套：流程上的已知断点见同目录 [`流程漏洞审查.md`](./流程漏洞审查.md)。

---

## 一、Next-Step 是什么

一个**多 Agent 软件工厂**本地工具（基于开源 pi-web 改造）：

- **项目即工作区** —— 每个项目独立，会话/Agent/产物互不串。
- **可自定义多 Agent** —— 为项目建专属 Agent（配模型/技能/工具/思考强度）。
- **可派发多 Agent 协作** —— 设个目标，让多个 Agent 串行接力产出文档。
- **产物的块级 Diff / 版本 / 按块确认** —— 文档型产物的每处改动可逐块审核、留版本。

特点：**本地单用户、纯文件落盘（无数据库/无登录）、并发会话 ≤ 3**。

---

## 二、5 步跑通主流程

```
①打开应用                      ②建/选项目              ③主对话开聊
  项目卡片墙        ── 点项目 ──▶  落到「主对话」  ── 输入 ──▶  Agent 回答
  （ProjectHome）                  （工作台 AppShell）
       │                                                          │
       │                          ④建 Agent / @转交                ▼
       └─────────────────────────  在主对话打「@」选 Agent   ⑤产物按块确认
                                    把历史+附件转交过去      右看 Diff · 左逐块 ✓/✗ · 留版本
```

> 最短路径：**建项目 → 进工作台 → 主对话里直接聊**就能用。Agent / 转交 / 产物确认是进阶。

## 三、功能逐个上手

### 1. 项目墙与项目（M6）
- **打开应用**先看到**项目卡片墙**，每张卡显「项目名 + 路径 + 创建于」。
- **建项目**：填名称 + 本地路径 → 入墙。
- **进工作台**：点项目卡。
- **回项目墙**：工作台内点「回到项目墙」。
- **删项目**：二次确认——注意文案，**只移除注册、不删磁盘上的 `.pi/` 数据**；重新加同一路径即恢复（本地单用户下是特性）。

> 小知识：首页与工作台是**同一个页面按「当前项目」二选一渲染**（无独立 URL，故没有项目深链）。

### 2. 主对话与左栏分组（M7）
- 进项目默认落到该项目唯一的**「主对话」**。主对话不预建空会话——**你在主对话发出第一条消息时，它才被自动认定为 main**。
- 左侧会话栏按归属分**三区**：**主对话** / **各 Agent 分组**（带 agent 名 + 色点 + 计数，可折叠）/ **其它会话**。
- 切项目不串显（各项目分组隔离）。

### 3. 建 Agent（M4 玻璃卡片）
- 顶栏 **Agents** 按钮（**需先选项目**才可点）→ 打开 AgentManager。
- 界面是**玻璃质感正方形卡片网格**，每卡只显真名 + 首字母色块。
- **点卡片** → 就地弹**浮层菜单**，三段：
  - **配置**：模型 / 技能 / 工具 / 思考强度，现场改即时保存（无独立「编辑」按钮）。
  - **起会话**：输入开场白 → 起该 Agent 一条会话（自动归入它的左栏分组）。
  - **删除**：二次确认显真名。
- **新建**：网格里点「**+**」空卡 → 填名称（必填）+ 配置。

> 字段速记：**角色描述**=这个 Agent 的职责/风格；**思考强度**=是否启用深层思考（高=更慢更细）；**工具**=允许它用的内置编码工具（读/写/bash 等）；**模型**留空=用全局默认。

### 4. 主对话 @agent 转交（M8）
- 在**主对话**输入框打 **`@`**（行首或空格后）→ 弹该项目 **Agent 待选列表**。
- 选一个 Agent → 弹「**转交内容勾选**」：默认勾【全主对话历史】+【已附文件】，可现场增减。
- 点「转交」→ 系统把内容打包成 `<context source="主对话">…</context>`（**保留 用户/助手/工具 角色标注**）+ `<file>` 附件，**起该 Agent 一条新会话**投递，自动切过去。
- 用途：让专门 Agent 接手主对话的上下文继续干。

> 注意：**`@` 只在「主对话」里生效**；在别的会话里打 `@` 没反应。每次转交起的是**新会话**（续接到 Agent 旧会话是后续增强）。

### 5. 上传文本文件（M2）
- ChatInput 左下**文件按钮**（和传图片同一个，图标是相册样）→ 选文件。
- 支持文本类白名单：`.md .txt .json .csv .log .py .ts .tsx .js .jsx .html .css .yaml .yml .xml .sql .sh`。
- 输入框上方出现「已附文件」标签（可点 ✕ 移除），发送时内容内联进消息。
- 单文件 >256KB 有「大文件耗 token」软提示（不阻断）。
- **PDF / Word / Excel 暂不支持**（选了会提示「暂不支持，建议转成文本」）。

### 6. 多 Agent 派发（Dispatch，V1）
- 设一个目标 → 派发面板让多个 Agent**串行接力**，每个产出落 `.pi/artifacts/<dispatchId>/` 下的 `.md`。
- 与 `@agent` 转交是**两套并存**机制：派发=自动接力多 Agent；@转交=手动把上下文甩给单个 Agent。

### 7. 产物：Diff / 按块确认 / 版本（V1·Iter D + M3 提示）
- 顶栏 **Artifacts** 按钮 → 选一个受管产物 → 右侧 **ArtifactPanel** 打开。
- 有改动时：正文**行内高亮**（绿=新增 / 红=删除 / 黄=修改），右上「N 处待确认」。
- **分工提示（M3）**：右侧顶部「→ 在左侧对话框逐块确认」，左侧卡片「改动全貌见右侧产物面板（按 `D` 看并排 Diff）」。
- **按块确认**：左侧 **PendingChangeCard** 逐块 **✓（确认）/ ✗（拒绝）**，或快捷键 Y/N/R/D；全部处理完才**写盘 + 生成新版本**。
- **版本管理**：版本下拉切换历史版（只读）；可**回滚**到某版（二次确认、生成新版）。

> 红线：**产物的任何改动都必须经「按块确认 → 写盘 → 新版本」**，不会被 Agent 直接覆盖。

## 四、原理分析：高亮 / Diff / 确认是怎么实现的？Agent 怎么用上？

> 先讲结论：**这些是 Next-Step 写死的应用功能，不是 Agent 实现的；而且设计上 Agent「不需要知道」它们——本该由系统在背后透明拦截。但这根「拦截线」目前还没接上（v2 主线），所以现在 Agent 实际还碰不到这套机制。**

### 1. 高亮 / 切块 / 确认 / 版本 —— 全是写死的应用代码

和 Agent 无关，是固定能力：

| 功能 | 由谁实现 | 在哪 |
|---|---|---|
| 把「旧内容 → 新内容」切成 add/del/mod 块 | LCS 切块算法 | `lib/domain/pending-change-service.ts` |
| 绿/红/黄 行内高亮、并排 Diff | 纯渲染函数 + 面板 | `lib/artifact-view/` + `components/ArtifactPanel.tsx` |
| 逐块 ✓/✗ 确认、写盘 | 卡片 + resolve API + 存储 | `components/PendingChangeCard.tsx` + `lib/domain/pending-change-service.ts` |
| 版本提交 / 回滚 | 领域服务 | `lib/domain/artifact-service.ts` |

这些代码任何时候都在、固定运行，不依赖某个 Agent「会不会用」。

### 2. Agent 怎么「产生」一处待确认改动？—— 靠透明拦截，Agent 无感知

**这是设计上最巧妙的一点：Agent 根本不用知道有「按块确认」，也不用学怎么用。**

- 起 Agent 会话时，系统**本应**给它一套**特制的 `write`/`edit` 工具**（`lib/pi/artifact-guard.ts` 的 `assembleArtifactGuardOptions`）。这套工具**外表和内核原版一模一样**，Agent 分辨不出（决策 D-D2-1：零工具漂移）。
- Agent 像平常一样用 `write`/`edit` 改文件。
- 但这套工具内部会**自分流**：
  - 改到**受管产物** → 不直接覆盖文件，而是**自动**读出当前内容、和新内容算出 Diff 块、存成一条「待确认改动」（`interceptManagedWrite`），**不写盘**；
  - 改普通文件 → 照常真实写盘。
- 于是：**Agent 无感知地「写」，你收到的却是「待你逐块确认的改动」**。审核权留在人手里，Agent 不用被教、也教不坏（它以为自己正常写完了）。

> 所以你问的「不同 Agent 怎么知道并用好这个功能」——**正确答案是：它们不需要知道**。功能藏在工具层、对 Agent 完全透明。这比「教每个 Agent 记得调某个确认 API」健壮得多——Agent 会忘、会绕过，那样红线就守不住了。

### 3. ⚠️ 现状：这套「特制工具」还没真正发给 Agent（v2 命门）

诚实地说，上面第 2 点目前**只到「机制写好 + 测过」，还没接线到真实会话**：

- `assembleArtifactGuardOptions`（造特制工具的函数）写好了、有单测，但**起 Agent 会话的代码从没调用它**（全仓库运行时零调用者）。它的用法注释写得明明白白——要把返回的 options 传进 `createAgentSession`——但这一步没人接。
- 后果：现在 Agent 会话拿到的还是**普通 `write`/`edit`**（直接写盘），不是守卫版 → Agent 写产物**不会**自动变成待确认改动。
- 当前按块确认 UI 能显示的，只有**别的途径造出来的** pending（手动、测试 fixture，或将来接线后 Agent 写触发的）。
- **派发（Dispatch）产物**又走另一条路：直接落成普通 `.md`，也不进受管系统。

**一句话**：零件全造好了（拦截器 + 切块 + 高亮 + 确认 UI + 版本），就差「起会话时把拦截器装上」这最后一根线——把 `assembleArtifactGuardOptions` 的 options 传进 `createAgentSession`。这是 **v2 主线**的头号任务，详见 [`流程漏洞审查.md`](./流程漏洞审查.md) §P0。

## 五、新手须知（当前限制，别踩）

| 主题 | 现状 | 说明 |
|---|---|---|
| **按块确认的来源** | 当前主要审核**派发/已生成**的 pending；真实 Agent 实时写产物**暂不会自动触发**按块确认拦截 | 这是 v2 主线（拦截接线）——详见 [`流程漏洞审查.md`](./流程漏洞审查.md) §P0 |
| **操作回执** | 转交 / 起会话 / 确认 / 回滚成功后**多数没有明显提示**（无 toast） | 成功了请耐心等一下界面刷新，别重复点 |
| **`@` 转交** | 只在**主对话**生效、无输入提示 | 记住：主对话输入框打 `@` |
| **Agent 会话分组延迟** | 起 Agent 会话后，左栏分组可能**延迟几秒**才出现 | 正常（后台落盘+刷新），不必手动刷新 |
| **文件类型** | 仅文本类；PDF/Word/Excel 不支持 | 转成文本再传 |
| **失败提示** | 转交失败 / 模型回退 / 技能缺失只打到**控制台**，界面无提示 | 功能异常时可开浏览器 Console 看 |
