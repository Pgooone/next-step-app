## M2 · chat-file-upload（功能#4）

**改动文件**：`components/ChatInput.tsx`（前端，**不碰内核 / pi-ai**）

**对外行为**
- 文件选择器 `accept`：图片 `image/*` 之外 **新增**文本类扩展名白名单 🔧：`.md,.txt,.json,.csv,.log,.py,.ts,.tsx,.js,.jsx,.html,.css,.yaml,.yml,.xml,.sql,.sh`（可扩展）。
- 选中后分流：
  - 图片 → 走**现有多模态通道**（`processImageFiles`，不动）。
  - 文本类 → `FileReader.readAsText` 读出文字，按内核官方格式拼成 `<file name="文件名">\n<内容>\n</file>` 文本块，附加到本条消息文本里再发送。浏览器上传无绝对路径，`name` 用**文件名**；与内核 `@file` 的**绝对路径** name 语义不同，本功能可接受。
- 发送前在输入框上方显示「已附文本文件」小标签（可移除），与图片预览并列。

**关键逻辑**
- 不设大小硬上限（用户决策）；🔧 单文件 > 256KB 时给 inline 软提示「大文件会消耗大量 token」，不阻断。
- 二进制 / 非白名单文件：提示「暂不支持，建议转成文本」，不静默失败。

**验收要点**：能选并发送文本类文件、模型读到内容；图片功能不受影响；大文件有软提示。

## M5 · session-agent-mapping（功能#5 地基·承重墙）

**问题**：会话数据无 `agentId`（`lib/types.ts:174-184`），且**不能改内核会话文件** → 领域层另存「会话 ↔ 归属」映射。

**改动文件**（新增）：
- `lib/domain/session-agent-map.ts`（领域逻辑）
- `app/api/projects/[id]/session-map/route.ts`（读写）
- `lib/stores/useSessionMapStore.ts`（前端状态）

**数据结构** 🔧（存盘：项目根 `<cwd>/.pi/ns-session-map.json`，与项目同处、纯文件）
```ts
interface SessionMap {
  mainSessionId: string | null;        // 该项目固定「主对话」会话 id（无则懒创建/取首个）
  bySession: Record<string, string>;   // sessionId -> agentId（某会话属于哪个 agent）
}
```

**对外接口**
- `GET  /api/projects/[id]/session-map` → `SessionMap`
- `PATCH /api/projects/[id]/session-map` → 设 `mainSessionId` 或 增删 `bySession[sessionId]`
- 领域层：`getOwner(cwd, sid)` / `setOwner(cwd, sid, agentId)` / `getMain(cwd)` / `setMain(cwd, sid)`

**关键逻辑**
- 映射是**附加元数据**，不写进内核会话文件 → 不违反「不改内核」红线。
- 删除清理 = **惰性清理**：读映射时丢弃已不存在的会话项（兜底外部直接删 `.jsonl` 的情形），不挂 DELETE 链路。

**验收要点**：能持久化「会话属于某 agent / 是主对话」；不动内核会话文件；读映射时丢弃已不存在的会话项（惰性清理，映射不残留）。

## M8 · at-agent-transfer（功能#5.5，依赖 M5 + M7）

**改动文件**：`components/ChatInput.tsx`（@ 触发 + 勾选）+ 转交逻辑。

**交互** 🔧
- **主对话**输入框输入 `@` → 弹**待选 agent 列表**（读该项目 agent 档案）。
- 选中 agent → 弹「**转交内容勾选**」：默认勾选【当前主对话全部历史】+【已附文件】，可现场增减（用户决策：全历史 + 附件 + 发送时现勾）。
- 确认 → 组装成一条消息，**投递到该 agent 单独会话**（无则按需起一个并经 M5 标记）；可选在主对话留「已转交给 @X」回执。

**转交载荷** 🔧
- 历史：主对话消息序列化拼进目标会话首条消息 `<context source="主对话">...</context>`，**保留角色标注**（user/assistant/工具）。
- 文件：沿用 M2 的 `<file>` 内联格式（浏览器上传无绝对路径，`name` 用文件名；与内核 `@file` 的绝对路径语义不同、本功能可接受）。

**关键逻辑**（D-V1.1-04：转交 = 起新会话）
- 投递机制 = **起新会话**：`POST /api/projects/[id]/agents/[agentId]/session`，组装好的载荷（`<context>` + `<file>`）作 `firstMessage`；复用 `handleAgentSessionStarted` 切到该会话并接 SSE，新会话经 M5 标记归属。
- 每次 `@agent` 转交起一条该 agent 的**单独**新会话（复用既有会话需按 `sid` re-attach，列为后续增强，见 D-V1.1-04）。
- 是**新轻量功能**：不复用 Dispatch（保留不动）、不在主对话就地回答。

**验收要点**：主对话 `@` 唤出 agent；转交内容经 `POST /session` 进入该 agent 单独新会话；默认带全历史+附件、可勾选；**序列化保留角色标注（user/assistant/工具）且在目标会话可见**；**确认 @agent 与 Dispatch 并存、互不干扰**。
