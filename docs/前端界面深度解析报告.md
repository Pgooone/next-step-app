## 0. 先记住一句话：这是「一个页面」的应用

很多软件是「一页一页跳」的（点登录 → 跳到首页 → 跳到设置页，地址栏一直在变）。**Next-Step 不是。**

- 整个应用**只有一个网址 `/`**（`app/page.tsx:4` 只渲染了一个叫 `<AppShell/>` 的总组件）。
- 你点的几乎所有按钮，**都不会跳转网页**，只是在同一页上「弹出一个浮层」或「把右边那块内容换掉」。
- 地址栏顶多在后面加一个 `?session=xxx`，用来记住「你当前打开的是哪个对话」，仅此而已。

所以理解这个界面最省力的心法是：**它长得像 VS Code / 一个聊天软件 —— 左边管文件和入口、中间和 AI 聊天、右边看文件或文档。**

```
浏览器地址栏：  http://localhost:30141/        ← 永远是这个，不变
页面内部：     [ 左栏 ] [ 中间聊天 ] [ 右栏 ]   ← 三栏，左右都能收起
弹出层：       Models / Skills / Agents / Dispatch / Artifacts  ← 浮在最上面的窗口
```

> 📖 **几个全文会反复出现的词，先解释清楚：**
> - **下拉框（select）**：点一下会掉下来一列选项让你挑，挑完收起来。本文里凡是说「下拉」都指这种。
> - **二次确认**：危险操作（删除、回滚）点一下不会立刻执行，会先变成「确认 / 取消」两个按钮，逼你再点一次，防手滑。
> - **开关（toggle）**：一个左右滑动的小滑块，亮起=开、灰着=关。
> - **`/api/...`**：**不是网页，是「后端服务接口」**。你看不见它，是界面在背后偷偷打的「电话」，用来存数据/取数据。
> - **SSE（服务器推送）**：一种「后端有新内容就主动推给前端」的实时通道，聊天逐字蹦出来、文件改了自动刷新都靠它。
> - **轮询（polling）**：前端「每隔几秒主动问一次后端：好了没？」，派发进度就是这么刷新的。

## 1. 路由与页面：到底有几个「页面」？

| 路由（网址） | 文件 | 职责 |
|---|---|---|
| `/`（唯一页面） | `app/page.tsx:4` → `components/AppShell.tsx:27` | 整个应用的全部界面，三栏布局 + 5 个可弹出的窗口 |
| `app/layout.tsx` | `app/layout.tsx` | 不是页面，是「外壳」：定义全局字体、主题、把 `/` 套进 HTML 骨架 |

**结论：用户能「看到」的页面只有一个。** 没有「项目列表页」「设置页」「文件预览页」这些独立网页 —— 它们全都是同一页里的「区域」或「弹窗」。

## 2. 顶层骨架：AppShell 三栏布局

`AppShell`（`components/AppShell.tsx:27`）是「总装配工」，它把界面拼成从左到右三块：

```
┌──────────────┬───────────────────────────────┬──────────────────┐
│  左栏(可收起)  │  中间：顶部工具条 + 聊天区        │  右栏(可收起)      │
│              │                               │                  │
│ 项目/会话/文件 │  和 AI 对话的主舞台              │ 文件查看 或 产物面板 │
│ 底部5个入口    │                               │ （二选一，互斥）   │
└──────────────┴───────────────────────────────┴──────────────────┘
                                                  ↑右上角永远有个小按钮开关右栏
```

三个关键事实，务必先记牢：

1. **左栏和右栏都能折叠**（收起后给中间聊天腾地方）。左栏折叠按钮在顶部工具条最左（`AppShell.tsx:444`，鼠标悬停显示 "Show/Hide sidebar"）。
2. **右栏是「文件查看器」和「产物面板」二选一**，不能同时显示 —— 一旦你打开了某个「产物」，产物面板会**盖住**文件查看器（`AppShell.tsx:834`：只要 `selectedArtifactId` 有值就显示产物面板，否则才显示文件）。
3. **右上角那个固定小方块按钮**（`AppShell.tsx:847`，鼠标悬停显示 "Show/Hide file panel"）是用来开关右栏的，它永远在那里。

- 在你**当前选中的项目目录**里，开一个空白对话框。

### 3.3 「刷新」按钮（圆形箭头）

### 3.5 项目目录选择器（CWD picker）
- 和项目切换器**并排的第二个下拉**，列出最近用过的 5 个目录（`SessionSidebar.tsx:38` 的 `getRecentCwds` 只取最近 5 个）。
- 可选「Use default directory（用默认目录）」或「Custom path…（手填路径，回车校验）」。手填路径会调 `/api/cwd/validate` 校验目录是否存在。

> ⚠️ **新手最容易混的第一组概念 —— 「项目」≠「目录」：**
> - **项目切换器**管的是「已注册的项目」（有身份、有 id）。**只有选了项目，底部的 Agents / Dispatch / Artifacts 才能用。**
> - **目录选择器**管的只是「会话在哪个文件夹里工作」，跟「项目身份」无关。
> - 一句话：**项目 = 有户口的工作区；目录 = 随便指一个文件夹。** 它俩长得像，但权限完全不同。

### 3.6 会话列表
- 把当前项目目录下的会话列出来，支持**父子树形**（`SessionSidebar.tsx:68` 的 `buildSessionTree`）：从某条消息「分叉」出来的子会话会缩进、带分叉图标、可折叠。
- 每条会话：**点击 = 打开**；鼠标**悬停**会冒出「重命名（铅笔）」和「删除（垃圾桶）」，删除会就地变成 Delete / Cancel 两个按钮二次确认。
- 每条会话显示相对时间（`SessionSidebar.tsx:23` 的 `formatRelativeTime`：just now / 5m ago / 3h ago / 2d ago / 具体日期）。

### 3.7 文件浏览器 Explorer（`components/FileExplorer.tsx`）
- 树状列出当前项目目录的文件夹/文件。
- 点**文件夹** = 展开/收起（用到才加载子项）；点**文件** = 在右侧面板打开它。
- 悬停文件右侧出现一个「mention」按钮 —— 点它会把**该文件的相对路径**以反引号代码形式插进聊天输入框（`AppShell.tsx:131-133` 的 `handleAtMention`）。
- 这是目前界面上**唯一和「@」沾边的功能**，但它只是「插一段文件路径文字」，**不是「@某个 Agent」**（这点后面功能#5 会重点澄清）。

### 3.8 左栏最底部：5 个功能总入口（核心）
在 `AppShell.tsx:314-403`，从左到右 5 个图标按钮，点击各自**弹出一个浮层窗口**。下面这张表把每个按钮的「能不能点」的精确条件也列了（去源码核对过）：

| 按钮 | file:line | 何时**变灰点不动** | 点了弹出什么 | 用途一句话 |
|---|---|---|---|---|
| **Models** | `AppShell.tsx:316-329` | 永远可点（`disabled: false`） | ModelsConfig | 配置 AI 模型和密钥 |
| **Skills** | `AppShell.tsx:330-341` | 没有任何可用目录时（`!activeCwd && !selectedSession?.cwd && !newSessionCwd`） | SkillsConfig | 管理给 AI 的「技能说明书」 |
| **Agents** | `AppShell.tsx:342-354` | 没选项目时（`!currentProjectId`） | AgentManager | 管理自定义 AI 角色（Agent 档案） |
| **Dispatch** | `AppShell.tsx:355-368` | 没选项目时（`!currentProjectId`） | DispatchPanel | 让多个 Agent 接力协作产出文档 |
| **Artifacts** | `AppShell.tsx:369-381` | 没选项目时（`!currentProjectId`） | ArtifactPicker | 打开「受管产物」文档 |

> 变灰时按钮透明度降到 0.35、鼠标变成普通箭头点不动（`AppShell.tsx:393`）。这 5 个就是 Next-Step 的功能总开关，第 7 节会把每个弹窗里的控件全部拆开。

## 4. 中间·顶部工具条（`AppShell.tsx:443` 起）

从左到右：

- **折叠侧栏按钮**（`AppShell.tsx:444`）：收起/展开左栏。
- **主题切换按钮**（太阳/月亮，`AppShell.tsx:466`）：亮/暗模式切换，带从按钮位置扩散的动画。
- **以下三个只在「有打开的会话」时才出现：**
  - **Export（导出 HTML）**：把当前会话导成一个网页（跳到 `/api/sessions/[id]/export`）。会话还没存盘时是灰的。
  - **Branches（分支）**：下拉展示这个会话的「分支树」，可点节点切到别的分支（`BranchNavigator.tsx`）。
  - **System（系统提示词）**：下拉**只读**展示本次对话拼出来的「系统指令」（AI 看到的隐藏开场白）。
- **工具条最右**：会话统计 —— 输入/输出/缓存 token 数、花了多少美元 `$`、上下文占用百分比（悬停有详细 tooltip）。这些数字是聊天区算好后报上来显示的。
- **工具条下方**有时会冒出**黄色横幅「未配置模型凭证」**，带「去配置」（直接打开 Models）和「✕」关闭。它由健康检查接口 `/api/health` 驱动（`AppShell.tsx:54-58`）。

> 名词解释：**token** 是 AI 计费和「记忆容量」的最小单位，约等于「大半个英文单词 / 一个汉字」。**上下文占用百分比**就是「这次对话塞进 AI 脑子里的内容，占了它记忆上限的百分之几」，越接近 100% 越该「Compact 压缩」（见 7.6）。

## 5. 中间·聊天区（这是「买来的」基座能力）

聊天区由 `ChatWindow` + `ChatInput` + `MessageView` 组成，**逻辑几乎全在 `hooks/useAgentSession.ts` 里**，`ChatWindow` 只是个壳。

> 给新手的背景：聊天、文件查看、模型/技能配置这些是从开源项目 **pi-web「继承/买来」**的成熟能力；项目「自研」的是后面的项目/Agent/派发/产物那套（见第 7、8 节）。

### 5.1 空会话长什么样
大标题「π Pi Agent Web」+ 打字机滚动文案 + 版本号，下面就是输入框。

### 5.2 有内容时：上面消息流，下面输入区
**消息流（`MessageView.tsx`）：**
- **用户消息**靠右（蓝底气泡，支持图片、Markdown），悬停出现三个按钮 + 时间戳：
  - **Copy**（复制）
  - **Edit from here**（从这里改 —— 会在**同一个会话内部分叉**）
  - **New session**（从这里 fork 出一个**独立的新会话**）
- **助手（AI）消息**靠左：显示模型名、流式打字时显示预估 token 和速度（t/s），含可折叠的「**Thinking 思考过程**」块和「**工具调用**」块（点开看入参和返回，成功绿框/出错红框），底部有用量统计和复制按钮。
- 右侧贴边有个 `ChatMinimap` 消息缩略导航条（用来快速跳转长对话）。

> ⚠️ **新手最容易混的第二组概念 —— 「Edit from here」≠「New session」：**
> - **Edit from here**：在**当前这个会话里**改写、长出一条新分支（还是同一个会话，能用顶部 Branches 切回去）。
> - **New session**：把这里之前的内容**复制成另一个完全独立的新会话**。
> - 简记：**Edit = 同一个会话内换条路；New session = 另起炉灶复制一份。**

### 5.3 输入框 ChatInput（`components/ChatInput.tsx`）—— 逐个控件拆开讲

这是你每天用最多的一块。它分**两种状态**：AI 没在跑（idle）时一套控件，AI 正在跑（streaming）时换另一套。下面按区域 + 状态逐一列全。

**(A) 中间多行文本框**（`ChatInput.tsx:364-399`）
- 这是你打字的地方。**Enter 发送、Shift+Enter 换行**（`ChatInput.tsx:203-211`）。
- 支持**粘贴图片**（直接 Ctrl+V 贴图，`ChatInput.tsx:223-230`）和**拖拽图片**进来。
- 占位提示文字会随状态变：空闲时「Message…」；AI 在跑且支持插话时「Steer 立即注入 / Follow-up 排队…」；AI 在跑但不支持时「Agent is running…」（`ChatInput.tsx:378-383`）。
- 输入中文时按 Enter **不会误发**（代码专门处理了中文输入法的「正在拼字」状态，`ChatInput.tsx:192-201`）。

**(B) 文本框右侧的发送 / 插话按钮**

| 控件 | file:line | 何时出现 | 作用 |
|---|---|---|---|
| **Send（发送）** 蓝按钮 | `ChatInput.tsx:452-478` | AI **空闲**时 | 把消息发出去。文本框空且没附图时是灰的、点不动（`:454`）。 |
| **Steer（打断插话）** 黄按钮 | `ChatInput.tsx:403-425` | AI **正在跑**且支持 steer 时 | 「打断 Agent 当前运行，立即注入消息」（`:407` title）。即不等它说完，马上把你的话塞进去。 |
| **Follow-up（排队追问）** 紫按钮 | `ChatInput.tsx:426-449` | AI **正在跑**且支持 followup 时 | 「在 Agent 完成后排队发送」（`:430` title）。即等它这轮说完，再自动把你的话发出去。 |

> 注：AI 在跑时直接按 Enter，默认走 Steer（没有 steer 才走 followup），见 `ChatInput.tsx:205-207`。

**(C) 底部左下角：附图 + 模型下拉**

| 控件 | file:line | 选项 / 作用 |
|---|---|---|
| **附图按钮**（回形针/图片图标） | `ChatInput.tsx:486-515` | 点它弹出系统选文件框，可多选图片（`accept="image/*" multiple`，`:290-301`）。AI 在跑时变灰禁用。已附图时图标变成强调色。 |
| **模型选择下拉**（CPU 芯片图标 + 当前模型名） | `ChatInput.tsx:516-616` | 点开**按 provider（供应商）分组**列出所有可用模型（`:572-611`）。每组有个灰色大写的组名标题（仅当有多个供应商时显示，`:574`）。当前选中的模型左边有蓝对勾、加粗（`:603-605`）。点某个模型即切换；AI 在跑时整个下拉变灰禁用（`:525`）。 |

> 模型选项是怎么来的？优先用带供应商信息的 `modelList`，没有就退而用 `modelNames`（`ChatInput.tsx:235-244`）。分组逻辑见 `:247-252`。

**(D) 底部右下角（AI 空闲时）：思考强度 + 工具预设 + Compact + 声音**

**① 思考强度下拉（7 档）**（`ChatInput.tsx:624-713`）
按钮上是个灯泡图标 + 当前档位名。点开是一列，**每档右边都有一句中文解释**（`ChatInput.tsx:53-61` 定义）：

| 档位 | file:line（描述） | 含义 |
|---|---|---|
| **auto** | `ChatInput.tsx:54` | 「沿用 pi 默认设置」（不强制，交给内核默认） |
| **off** | `:55` | 「关闭推理」（不让它想，直接答，最快最省） |
| **minimal** | `:56` | 「最少推理」 |

## 8. 产物闭环：Next-Step 区别于普通聊天工具的「灵魂」

这是最该单独理解的一条主线，把多个组件串起来：

```
① 点左栏「Artifacts」→ ArtifactPicker 列出产物 → 选一个
        ↓（设置 selectedArtifactId）
② 右栏自动从「文件查看器」切成「ArtifactPanel」→ 看 AI 改了哪（行内高亮 / 并排 Diff）
        ↓（同一份数据）
③ 聊天区输入框上方冒出「PendingChangeCard」→ 你逐块 ✓/✗ 确认（或用 Y/N/全部✓/全部✗）
        ↓（一批改动全部处理完，服务端才写盘）
④ 生成一个新版本 vN+1（可在 ArtifactPanel 版本下拉里看历史、回滚）
```

**串起它们的是同一个数据仓库 `useArtifactStore`**（只认 `artifactId`，不认会话，`useArtifactStore.ts:99`），所以左边对话框确认、右边面板看改动，**拿的是同一份数据**。

**一条不可破的红线**（项目规矩）：对产物的任何修改，**必须经过「PendingChange → 按块确认 → 才写盘并生成新版本」**，不允许让 AI 或人直接改产物文件落盘。

## 9. 数据流：界面按钮怎么把数据送到后端

界面上的按钮**基本不直接联网**，而是走两条主干道：

### 主干一：管理类操作 → Zustand「数据仓库」→ 调 `/api/...`
（Zustand 是个「全局状态盒子」，你可以理解成「界面共用的数据中转站」）

| 你操作什么 | 经过哪个仓库 | 最终调哪个接口 |
|---|---|---|
| 项目增删切 | `useProjectStore` | `/api/projects*` |
| Agent 档案/起会话 | `useAgentStore` | `/api/projects/[id]/agents*` |
| 派发 | `useDispatchStore` | `/api/projects/[id]/dispatch` + 轮询 `/api/dispatch/[taskId]` |
| 产物内容/确认/版本/回滚 | `useArtifactStore` | `/api/artifacts/[id]*`（写操作带 If-Match 乐观锁，冲突返回 409） |

### 主干二：聊天相关 → `hooks/useAgentSession.ts`
- 发消息/停止/切模型/fork 等：POST 到 `/api/agent/[id]`；
- 接收 AI 的流式输出：用 `EventSource` 订阅 SSE `/api/agent/[id]/events`；
- 会话列表/历史：`/api/sessions*`；文件读取/监听：`/api/files/[...path]`。

### 少数组件直接调只读接口
FileExplorer→`/api/files`，SkillsConfig→`/api/skills*`，ArtifactPicker→`/api/projects/[id]/artifacts`，AppShell 健康检查→`/api/health`、目录校验→`/api/cwd/validate` 等。ModelsConfig 还会调一组 `/api/auth/*`（登录/密钥/供应商列表）和 `/api/models-config*`。

> 两种「实时更新」方式要分清：**SSE（服务器推送）**用于聊天流和文件实时同步（自动刷新）；**轮询（每隔几秒主动问一次）**用于派发进度（每 2 秒）。产物的版本/确认**没用 SSE**——是你自己操作完前端直接刷新（`useArtifactStore.ts:24` 注释 D-D5-2 选 A）。

## 附：给新手的「三句话避坑」

1. **「项目」和「目录」不是一回事** —— 选了项目才有 Agents/Dispatch/Artifacts；目录只管会话在哪干活。
2. **「Edit from here」是会话内分叉，「New session」是另起独立会话** —— 看清按钮再点，别把对话搞乱。
3. **产物 ≠ 普通文件** —— 产物有版本、有按块确认这套闭环；普通文件没有。右栏打开产物时会盖住文件查看器；改动只能通过 PendingChangeCard 逐块确认才落盘，谁都不能直接改产物文件。
