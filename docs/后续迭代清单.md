## 1. @agent 转交：复用 agent 既有会话（「一个 agent 一条连续会话」）

- **来源**：决策 `D-V1.1-04` 的选项 B；用户已确认「**先用 A（每次转交起新会话），B 留作后续增强**」。
- **本期（A）**：每次 `@agent` 转交都给该 agent 起一条**新**会话，转交载荷作 `firstMessage`。简单、复用 V1 已验收的 `POST /api/projects/[id]/agents/[agentId]/session` 端点、还顺带解决转交后接 SSE 流。
- **B 想要的**：永远续到该 agent 的**同一条**会话（一个 agent = 一条连续记录），更贴合「在自己会话接着干」的直觉。
- **为什么本期不做**：往「落了盘但未 live」的旧会话追加消息，真实代码里没有现成链路（`sendAgentCommand(sid, prompt)` 要求该会话已 live 注册）。
- **要做需补**：一个**按 `sessionId` 的 re-attach 端点**（重新 `registerInnerSession` + 注入 profile），再让 `@转交` 在「该 agent 已有会话」时走 `sendAgentCommand` 追加。
