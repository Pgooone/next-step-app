## 任务卡与落地

### T1 · 承重墙 · orchestrator 补 setOwner（Bug1 数据层，commit `bae6a95`）
- **orchestrator.ts**：`OrchestratorDeps` 增可注入钩子 `setOwner?`；`runDispatch` 内 `doSetOwner = deps.setOwner ?? setOwner`；在 `orchestrator.ts:152`（取到 `assignment.sessionId` 之后、**早于** timeout/aborted 判断）**无条件** `doSetOwner(projectRoot, result.sessionId, assignment.agentId)`。
- **写哪些**：completed / timeout / aborted 三类**都写**归属（决策③ = D-V1.2-23，失败 worker 也归 agent 名下）。三类前置失败（档案不存在 / `acquireSlot` 抛错 / `runWorker` 抛错）在 :152 之前 `return`、不写。
- **对照范式**：与交互式起 agent 会话路径 `app/api/projects/[id]/agents/[agentId]/session/route.ts:45` 的 `setOwner` 一致。
- **测试**：orchestrator.test.ts +6 spy 用例（断言 completed/timeout/aborted 写、三类前置失败不写）。
- 验收：lint 净 / tsc 零新增 / vitest 全绿。

### T2 · 承重墙 · 派发完成刷新分组（Bug1 接线，commit `c44e282`）
- **DispatchPanel.tsx**：新增独立回调 `onSessionsChanged`（Props 可选）。新增**独立** useEffect（独立 ref `notifiedSessionsTaskRef` 按 taskId 去重）：仅终态（`!selectIsActive(visibleTask)`）触发一次、调 `onSessionsChanged?.()`。**关键：不被 `producedArtifact` 门控**——与既有 `onArtifactsChanged`（被 `if(!producedArtifact)return` 挡、仅产受管文档才 fire）并存、各刷各的（D-R5-02）。coding / 纯文本 worker（artifactId 恒空、占绝大多数 dispatch）跑完也要触发分组刷新。
- **AppShell.tsx**：新增 `handleDispatchSessionsChanged`——复用 `handleSessionCreated` agent 分支同款有界重试 `tick`（≤8 次 / 700ms），以 `before` = 当前 `bySession` 键数为基准，`map.bySession` 键数 `<= before` 表示还没拿到新归属就继续 tick；每次并 `setRefreshKey` 重拉 `/api/sessions`（让 worker 会话落盘后进侧栏列表）。DispatchPanel 挂载 +`onSessionsChanged={handleDispatchSessionsChanged}`（无新增 import）。
- 验收：lint 净 / tsc 零新增 / vitest 423 全绿。

### T3 · 进项目恢复主会话（Bug2，commit `b06e447`）
- **main-session.ts**：新增纯函数 `pickSessionToRestoreOnEnter(map, urlSessionId)`——`urlSessionId` 非空直接返回（URL 深链优先）；否则返回 `map.mainSessionId`；皆空 → null（走默认 cwd / 新建态 T4）。不查会话是否真实存在（调用方据返回 id 在 `allSessions` 里 find，找不到再降级）。
- **SessionSidebar.tsx**：恢复 effect **下沉重写**（不在 AppShell 另开 restore 路径，D-R5-04），两阶段：
  - **① 会话恢复**（`restoredRef` 守护、仅尝试一次）：`effectiveMap = selectMapForProject(map, loadedProjectId, currentProjectId)` → `target = pickSessionToRestoreOnEnter(effectiveMap, initialSessionId)`。`target` 非空且 `allSessions` 就绪 → `find` → `found ? (setSelectedCwd + onSelectSession(found, true)) : onInitialRestoreDone?.()`（找不到 = 主会话已删 / 陈旧 → 降级默认 cwd、仅 URL 深链才调 `onInitialRestoreDone`）；`target` 空 → `mapReady = loadedProjectId === currentProjectId`，**未就绪且有 currentProjectId 则 `return` 等 map**、就绪则标 `restored`。
  - **② 默认 cwd**：`if (selectedCwd !== null) return`；否则 `selectedCwdProp` 优先、回退 `getRecentCwds`。
  - **关键 wait-gate（D-R5-03）**：主会话恢复要读 `mainSessionId`，而 map 由 AppShell 异步 refresh；未就绪时 `selectMapForProject` 回退空 map（mainSessionId=null）。若不等，会先错过主会话恢复、误落默认 cwd 设了 `selectedCwd`，再被阶段②顶部 `selectedCwd !== null` 永久挡住。`map` 在 deps 里，refresh（本地文件读 ms 级）后 effect 重跑、gate 放行；URL 深链不依赖 map。
  - `currentProjectId` + `currentRoot` + `map/loadedProjectId` 三行订阅**上移**到恢复 effect 之前（原在中部 grouped 计算处；上移后无重复声明，下方 grouped useMemo / JSX 仍引用）。
- **测试**：main-session.test.ts +3 用例（URL 优先 / 无 URL 取 mainSessionId / 皆空 null）。
- 验收：lint 净 / tsc 零新增 / vitest 426 全绿（423→426）。
- 备注：纯函数签名为 `urlSessionId: string | null | undefined`（非 `string | null`）——`SessionSidebar` Props `initialSessionId?: string | null` 即含 `undefined`，调用点直接传不 coerce，须 `undefined` 入参才过 tsc（lead 已 commit 此签名）。

### T4 · 首次进新项目 = 主对话新建态（Bug2，无代码，D-R5-05）

- **守 D-V1.1-09**：临时 id 客户端懒落盘、不调 `startProfileSession` / 后端预建、不占并发槽。

## 真浏览器统一验收（PASS，lead 亲看截图）
- **Bug1（AC-1）**：coding 派发 done、关面板**不手动刷新**等 ~7s（T2 重试窗口）→ 左栏自动出 2 个 agent 分组（编码甲 / 编码乙各 count=1）、`bySession` 2 条、不堆「其它会话」。
- **Bug2 T4（AC-6）**：空项目进入 = 新建态可输入、发消息前无该 cwd 会话。
- **Bug2 T3 恢复（AC-3）**：发首条成主会话 → 再进**自动恢复**（主对话区高亮 + 中间区显示完整对话）、url 无 `?session=`。
- **Bug2 T3 降级（AC-5）**：删主会话（`mainSessionId` 陈旧）→ 再进新建态、不崩。
- **pageErrors（AC-7）**：无 JS 崩溃（仅 console 层 403 = 临时项目 `/api/files` + 会话切换竞态 fetch 失败，已知无关）。

## 已知限制 / 接受项（见 QA + ADR）
- 主会话不加保护、可像普通会话删（D-V1.2-22）；删后再进走新建态、发首条重新认定。
- 失败 / 超时 dispatch 会话也归 agent 名下、计入计数，不做成功 / 失败视觉区分（D-V1.2-23）。
- doc-mode dispatch worker re-attach 续聊得 7 工具集（含 propose_edit）vs headless 跑时 6 工具集，非对称、文档化接受（D-R5-06，机制零改）。
