## 用户报的两个问题

1. **Bug1**：dispatch 派发的多个 agent，各自的会话没按 agent 分类、在左侧会话列表**全堆在「其它会话」区**（期望每个 agent 的对话归到该 agent 名下）。
2. **Bug2**：希望项目打开（及从项目墙进入）时**自动起一个「主会话」并默认恢复**，区分「主对话（人自己的）」与各 agent 派发会话（现状进项目是「暂无会话，点上方 New 新建」空态）。

## 根因（lead 已 file:line 复核，可信）

- **Bug1 = 数据层缺一行，非展示层**：分组 UI 早已完备——`lib/session-grouping.ts:38-60 groupSessionsByOwner` 按 `ns-session-map.json` 的 `SessionMap{mainSessionId, bySession:{sid→agentId}}` 切「主对话 / 各 agent 分组 / 其它会话」三区，`SessionSidebar.tsx`（AgentSessionSection :904-981）已渲染色点+名+计数。**唯一缺口**：`lib/domain/orchestrator.ts:152` 派完 worker 只写 `assignment.sessionId = result.sessionId`，**从不调 `setOwner` 写 bySession**（全文件无 `setOwner`/`session-agent-map`）。对照正确范式 `app/api/projects/[id]/agents/[agentId]/session/route.ts:45` 的 `setOwner(projectRoot, result.sessionId, agentId)`——dispatch 漏的就是这一行的等价物。领域 API `setOwner`/`setMain`/`sessionsForAgent` 在 `lib/domain/session-agent-map.ts` 现成。
- **Bug2 = 进项目不恢复/不预置主会话**：`AppShell.tsx:83-94` 项目切换只 refresh(map)+refresh(agents)，不读 `mainSessionId` 恢复、不预置。主会话靠 `lib/main-session.ts:13-20 pickMainOnSessionCreated`「首条普通消息落地后懒认定」（**D-V1.1-09 明令不预建空会话**：撞 B4 幻影空会话坑、违 pi-web「首条消息才落盘」）。唯一恢复源是 URL `?session=`（从项目墙进入被 `AppShell.tsx:266-269 handleBackToHome` 清掉）→ 落空态。

## 用户拍板（三点，已定 A/A/A）

1. **进项目恢复**：A — 默认恢复该项目主会话（`mainSessionId`）；无主会话则中间区进入「主对话新建态」。URL `?session=` 深链优先级高于此默认恢复。（核实：系统无「上次浏览会话」记忆，故不引入。）
2. **主会话保护**：A — 不加保护，主会话可像普通会话一样删；删后再进项目走新建态、发首条消息重新认定。（不过度设计。）
3. **失败会话归类**：A — dispatch 失败/超时的 worker 会话**也归到 agent 名下、也计入分组计数**，不做成功/失败视觉区分（便于回看哪个 agent 跑挂；改动最小）。
