## 关键结论 / 教训

- 首页版本机制本就完好（next.config 构建期自动读 package.json + node_modules），缺口只是 version 从未 bump → 方案 B 约定补上、机制零改。
- **单 agent 调研报告 over-claim**：把一个 no-op（rpc）+ 一个可选功能（markdown）都包装成「已验证有用的修复」，还把 rpc 修复版本误归 0.6.17。靠 ultracode 对抗式 spike（npm pack 负对照 + 零扩展核实）+ lead file:line/npm pack 亲验才戳破。→ 承重事实必须对抗式核实 + lead 亲验。
- pi 内核 0.80.x 暂不升：含 pi-ai 入口重组破坏性变更，连上游 pi-web 0.6.18 都仍钉 ^0.79.10。详见 `UPSTREAM.md`。
