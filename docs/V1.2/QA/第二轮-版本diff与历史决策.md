## D-V1.2-10 · 需求1 版本间对比的「基准」怎么定

- **背景**：看某历史版时，行内 diff 要跟谁比？现有版本下拉只能选单个版本。
- **可选项**：
  1. **相邻对比 vN vs v(N-1)**：选中版对比紧邻上一版 = 「这一版引入了什么改动」。复用现有单下拉、改动最小。
  2. 任选两版互比：自由选 base/target，更灵活，但需第二个选择器、改 store 状态形态、工作量更大。
- **推荐**：选项 1（相邻）——契合用户原话「看清每个版本不同的位置和内容」，最小改动。
- **谁拍**：用户。
- **最终选择**：**1·相邻对比 vN vs v(N-1)**。v1（首版）无前驱 → 显示只读全文 + 「首版，无对比基准」提示。

## D-V1.2-11 · 需求1 行内 diff 如何与现有「看历史版（纯只读）」共存

- **背景**：现状 ADR **D-D5-4** 规定「看历史版 = 纯只读全文、无 diff」（祖先设计 `docs/设计决策记录.md`）。需求1 要在看历史版时显示 diff，与之冲突。
- **可选项**：
  1. **改造 viewingHistory 分支**：选历史版即显「该版 vs 上一版」行内 diff（改动小，但改变「看历史版=只读快照」语义、**推翻 D-D5-4**）。
  2. 新增并列「对比」视图模式：保留只读全文不变、另加一个入口（语义清晰但多一个入口、交互略重）。
- **推荐**：选项 1，并保留「回到只读全文」小开关作逃生口；在本 QA + ADR 记录推翻 D-D5-4。
- **谁拍**：用户。
- **最终选择**：**1·选历史版即显 diff**（推翻 D-D5-4，保留只读全文逃生开关）。

## D-V1.2-12 · 需求2 diff 历史的主入口放在哪里

- **背景**：用户原话「不知道在哪里新增记录会好一些，至少要更易读，甚至能在行内回看」。三个候选都**零新增存储、都能行内回看**（每版全量快照永不覆盖，相邻两版 diff 可重算）。
- **可选项**：
  1. **ArtifactPanel 内「Diff 历史」时间线 Tab**：集中清单（v_n · note · 时间 · 作者），点条目就地展开行内 diff。易读性最高、最契合「记录放哪里」。
  2. 版本下拉选版即看 diff：改动最小，但记录「藏在下拉里」、不像清单一目了然。
  3. 左栏文件树二级展开：入口在文件树、发现性好，但左栏窄、列表拥挤、回看要跳右栏（非原地行内）。
  4. （另：diff-log 侧车持久面板 = 见 D-V1.2-13 的「审计」深度，需新存储）
- **推荐**：选项 1（时间线 Tab）为主，顺带实现选项 2（选版即看）—— 两者共用同一渲染器、零新存储。
- **谁拍**：用户。
- **最终选择**：**1·ArtifactPanel 内「Diff 历史」时间线 Tab**。

## D-V1.2-13 · 需求2 diff 历史要记到多深（决定是否新增存储）

- **背景**：确认（按块确认）后，承载「改了哪些块 / 谁改的 / 确认或拒绝了哪些块」的 PendingChange 会被删除（`pending-change-service.ts:380` remove），只剩版本全量快照。故「每次改了什么内容」可零成本重算，但「逐块确认决策 / agent 身份」重算不回来。
- **可选项**：
  1. **只看每次内容改动**：从版本快照重算相邻版 diff，**零新增存储、零红线风险**，覆盖绝大多数回看诉求。
  2. 额外记审计轨迹：新增 diff-log 侧车，保留「哪个 agent 提议 / 确认或拒了哪些块」。需新存储 + 写路径 + 红线豁免定性 + 新 API，且**只能从今往后记、补不回已发生历史**。
- **推荐**：选项 1。审计（选项 2）作可选增强，待用户明确要再做。
- **谁拍**：用户。
- **最终选择**：**1·只看每次内容改动（零存储重算）**。审计侧车（C）本轮不做。

## D-V1.2-14 · 需求2 时间线点开后怎么显示

- **背景**：用户追问「选择 diff 历史后，是单独的二级浮窗，还是直接覆盖正文显示页」，并要求可视化简图对比。关键约束：面板**已有第七轮 A1 的全屏 gsap 悬浮浮层**。
- **可选项**（含完整真实工具栏：版本下拉 / 删除 / 引用到对话框 / 行内│查看Diff）：
  1. **甲·时间线铺满主体区（覆盖正文）**：点工具栏 `[Diff历史]` 按钮高亮 → 主体区整体切成时间线、覆盖正文；点条目手风琴就地展开行内 diff；时间线模式下隐藏 `[行内│查看Diff]` 段。与现有视图切换同一套逻辑、**无新浮层、不撞 A1 全屏浮层**、最 surgical。
  2. 乙·二级浮层框：点 `[Diff历史]` 弹独立浮层盖住正文+工具栏，点 ✕ 关闭。能快速开关，但与 A1 全屏浮层叠加有层级/手势冲突风险、复杂度高、窄面板更挤。
- **推荐**：选项 1（甲）——避免与 A1 浮层冲突、改动最小、手风琴满足「行内回看」。
- **谁拍**：用户（经两轮简图澄清后拍板）。
- **最终选择**：**1·甲·时间线铺满主体区**。`[Diff历史]` 按钮放版本下拉右侧、常驻可见；再点按钮或选版本下拉即回正文。
