## D-V1.2-1 · V1.2 工作区搭建方式

**背景**：用户因「V1.1 过于臃肿」（累积约 8.7M 验收截图 + 多轮历史 docs）新建 `next-step-V1.2`，但其中只有 `ui原型参考/`（sf-mini 原型 board.html / index.html），**无 app 源码、非 git 仓库**。要做视觉重构，必须先把可运行的 app 代码搬进 V1.2。

**可选项（全列）**：
- **A 精简副本 + 保留 git 历史（推荐）**：拷贝 v1.1 app 源码 + 配置 + docs，连同 `.git`（新建分支 `v1.2`、沿用远端 `Pgooone/next-step-app`）；剔除臃肿（node_modules 重装、`.next`/`*.tsbuildinfo`、5 个验收截图目录）。历史 / blame 完整，工作区干净。
- **B 精简副本 + 全新 git**：同样精简代码，但 `git init` 全新历史、干净起点；丢失 round1~7 提交历史与 blame。
- **C 完整镜像 v1.1**：原样整盘复制（含截图、node_modules、全部历史）；不丢任何东西，但没解决「臃肿」、只是又复制一份。

**推荐 + 理由**：A。保留可追溯性（blame / 历史对理解既有代码至关重要），同时达成「精简」诉求，且沿用远端便于后续直接 push。

**谁拍**：用户（AskUserQuestion）。

**最终选择**：**A**。
- 落地：因 `rsync` 不可用，改用 `tar` 管道带排除拷贝（完整保留 `.git`：历史 + origin=github + master/v1.1 两分支 + 未跟踪的有用文件如 sf-mini 反推文档；只剔除截图 / 构建产物 / node_modules）。建分支 `v1.2`，`npm install`（1224 包、43s）。结果：精简 12M（不含 node_modules），`master`=`v1.1`=`v1.2`=`e5f97d4`、origin=`Pgooone/next-step-app`、当前在 `v1.2`。

## D-V1.2-2 · agent 卡片视觉方向

**背景**：参考的 sf-mini（index/board.html）是**深空 + cyan/violet 霓虹**的纯深色玻璃风；而我们 app 是**亮/暗双主题**的生产力工具。重构 agent 卡片时，视觉要走多「忠实」于 sf-mini？这决定整个工作流的产出方向。

**可选项（全列）**：
- **主题自适应玻璃（推荐）**：把参考的玻璃质感、状态富信息、渐变强调「翻译」进我们现有 CSS 变量体系，亮/暗双主题自适应，与 app 其余部分协调。质感与信息量都升级，但不割裂。
- **深色指挥舱**：Agent 管理面板整体换成 sf-mini 的深色玻璃 + cyan/violet 霓虹，作为独立「指挥舱」表面，无视全局亮/暗主题。最贴近参考，但与亮色 app 其余部分风格割裂。
- **忠实照搬 sf-mini**：尽量 1:1 还原 `#070b16` 深空背景、径向辉光、霓虹配色，全局也转向深色风。改动面最大，牵动 app 整体观感。

**推荐 + 理由**：主题自适应玻璃。在「贴近参考的高级感」与「与现有 app 协调、不返工其余界面」之间取最佳平衡；复用既有 `.glass-card` 与 CSS 变量，落地风险最低。

**谁拍**：用户（AskUserQuestion，附 ASCII 预览对比）。

**最终选择**：**主题自适应玻璃**。

## 设计阶段拍板（2026-06-22，基于 ultracode 设计探索工作流的推荐 + 开放选项，说人话重问后用户拍板）

> 来源：设计探索工作流 `wf_3d884982-4fc`（研读 sf-mini 参考+现状 → 3 角度方案 → 对抗评审综合）产出推荐设计 + openChoices；lead 把技术黑话翻成"你实际会看到什么"重问、用户拍板。

### D-V1.2-4 · 卡片信息密度

**背景**：当前卡片只显「首字母色块 + 名字」，单卡信息收益近零；agent 档案有 role/model/mode/skills/tools/thinking 等真实字段被埋进二级菜单。卡面露多少？

**可选项（全列）**：
- **多一点·名片式**：名字 + 模式徽章 + 模型 + 角色摘要 + 技能/工具/思考计数（6 字段）。不点开就一目了然。代价：卡片高度随角色长短略有参差。
- **极简·清爽**：只头像 + 名字 + 角色一句话 + 文档/编码标记，其余进二级菜单。更清爽等高；但想对比"谁配了什么"得逐个点开。

**推荐 + 理由**：多一点·名片式——匹配用户「参考其内容展示方式」诉求，把被埋的真实字段提到卡面。

**谁拍**：用户。**最终选择**：**多一点·名片式（6 字段）**。

### D-V1.2-5 · 模式（文档型/编码型）信号形态

**背景**：mode（doc/coding）是卡上唯一带强语义的二元字段，怎么标？

**可选项（全列）**：
- **文字徽章「文档/编码」**：彩色文字小标签（编码=琥珀警示 / 文档=中性）。可读、看不清颜色也认得字。
- **纯色圆点（蓝/黄）**：右上小圆点 + 悬停释义。更简洁，但需悬停才知义、色弱不友好。

**推荐 + 理由**：文字徽章——可读性/可达性更好。

**谁拍**：用户。**最终选择**：**文字徽章「文档/编码」**。

### D-V1.2-6 · 头像是否加品牌渐变光边

**背景**：头像现为哈希纯色块 + 首字母；参考原型 logo 用 cyan→violet 品牌渐变。

**可选项（全列）**：
- **加渐变光边**：头像外加 1.5px cyan→violet 细光边。更精致、有高级感、呼应参考原型。
- **维持纯色块**：朴素、改动更小、少一组渐变 token。

**推荐 + 理由**：加光边——质感升级、呼应参考。

**谁拍**：用户。**最终选择**：**加渐变光边**。

### D-V1.2-7 · 本增量范围（是否含 DispatchPanel）

**背景**：「派发任务」界面（DispatchPanel）选 agent 是**另一套组件**（纵向列表行、非网格卡、自带硬编码选中色），不复用 AgentCard。另 AgentManager/Dispatch 有 4~5 处硬编码 `rgba(37,99,235,0.x)` 选中色（技术债）。本期一并统一吗？

**可选项（全列）**：
- **只做 AgentManager 卡片**：聚焦、稳、快、验收面小。派发界面留下次。
- **一并统一 DispatchPanel + 收技术债**：派发界面选 agent 列表也升级同款玻璃 + 模式徽章，并把 4~5 处硬编码 rgba 收敛成 `--accent-soft`（亮暗自适应）。视觉更一致，但改动/真浏览器验收面更大。

**推荐 + 理由**：lead 原推荐「只做 AgentManager」（聚焦、降风险）。

**谁拍**：用户。**最终选择**：**一并统一 DispatchPanel + 收技术债（扩大范围）**——用户取视觉一致性优先。

### lead 默认（小项，未单独问；详细设计据此定稿）

- **model 位置**：放身份行第二行（与名同列），页脚只放技能/工具/思考计数。
- **模式文案口径**：卡面徽章用短名「文档/编码」（省空间），与二级菜单全称「文档型/编码型」语义对齐。
- **字段渲染条件**（防误导/防噪音）：`tools` 计数仅 coding 模式且 >0 显示（doc 模式 tools 被内核忽略）；`thinking` 仅非 off 显示；role/计数为空则整段隐藏。

### D-V1.2-8 · 第一轮卡片视觉精修（用户看截图后反馈，2026-06-22）

**背景**：第一轮卡片重构（批次1+2）真浏览器功能验收全 PASS、已 commit。用户看亮/暗放大截图后反馈视觉问题，并就玻璃强度拍板。

**玻璃强度·可选项（全列）**：
- **A 保持现状（精致克制）**：玻璃靠品牌环 + hover 给质感、静态克制。
- **B 强化玻璃**：极光更明显 / 模态加环境底 / 描边更亮，更"显眼"。

**推荐 + 理由**：lead 看放大图后倾向 A（品牌环 + hover 已撑起质感）。

**谁拍**：用户。**最终选择**：**B 强化玻璃**。

**用户同时反馈的视觉缺陷（lead 亲看截图确诊、本轮一并修）**：
- 字号大小不一（10/11/11.5/13 散乱）→ 统一阶梯 16/13/12/11。
- 卡片分布乱 / 高度参差（role 2 行致忽高忽低）→ role 改 1 行、卡片等高。
- DispatchPanel agent 卡片非常乱（旧 flat 灰行、与新卡片割裂）→ 玻璃化对齐（批次3）。
- 玻璃显 flat 真因（lead 诊断）：玻璃卡浮在纯白/纯深模态上、无背景可 frost → 模态加极淡环境底。

### D-V1.2-9 · 改用 ui-ux-pro-max、转 Swiss 极简专业方向（2026-06-22）

**背景**：第二轮玻璃精修后，用户要「用 `ui-ux-pro-max` 技能、参考现有卡片 + Next-Step 页面风格，选一份更专业的 UI/UX 改造 agent 卡片 / dispatch / 管理界面」。技能 `--design-system` + style 检索给出的「专业」答案 = **Swiss Minimalism**（`Best For: Enterprise apps, dashboards, SaaS, professional tools`：高对比 / 精排版 / 严谨留白 / **克制阴影渐变** / 单一强调色 / 清晰焦点态 / 亮暗双主题 WCAG AAA）。

**方向·可选项（全列）**：
- **Swiss 极简**：收掉极光 / 厚玻璃，转 Linear/Vercel 那种干净专业（精排版 + 严谨留白 + 高对比 + 细边 + 单蓝强调 + 清晰 hover/焦点）。技能推荐、最符合「专业开发者工具」。
- **玻璃为主·再抠专业细节**：保留玻璃主调（前轮选的 B），用专业规则打磨排版 / 间距 / 状态 / 图标。
- **混合**：极简骨架 + 头像环 / 选中态一抹玻璃点缀。

**推荐 + 理由**：Swiss 极简——技能数据库对「professional developer tool」的明确答案；真专业感来自排版 / 留白 / 对比而非装饰。

**谁拍**：用户。**最终选择**：**Swiss 极简**（方向从玻璃转向干净专业；**保留第一轮卡片信息结构、只换视觉执行**）。

落地见 `../第一轮-多Agent管理/第三轮-Swiss专业重做.md`。
