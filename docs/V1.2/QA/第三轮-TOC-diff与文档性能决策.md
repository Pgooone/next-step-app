## D-V1.2-15 · 需求A TOC diff 的视觉形态

- **背景**：版本对比时左侧目录要体现章节级 diff（新增/删除/修改）。用户经多轮简图澄清最终定形。
- **可选项**（均配色复用正文 diff 绿/红/黄）：A 标题左侧色点 / B 左侧色条 / C 右侧文字徽章 / D 整条变色底 / **E 改动行下方色横线**。
- **推荐**：B 或 E（轻、与正文 diff 配色一致）。
- **谁拍**：用户（三轮简图迭代后定）。
- **最终选择**：**E·改动章节标题下方一条「对应类型颜色」的横线**（border-bottom：绿=新增 #4ade80 / 黄=修改 #eab308 / 红=删除 #f87171），**线本身着色、无圆点/图标**，改动行与上下相邻行**留出间距**（色线清晰归属该行、不拥挤）。

## D-V1.2-16 · 需求A 生效范围

- **背景**：TOC diff 在哪些态生效？版本对比（viewingHistory）/ Diff 历史时间线 / pending diff。
- **可选项**：① 仅 viewingHistory（改动面小）；② + 时间线手风琴 mini-TOC（改动大、易过载）；③ + 时间线放开全局 TOC。
- **推荐**：① 仅 viewingHistory——base/target 现成、零额外接线、覆盖核心诉求。
- **谁拍**：lead 代定（用户授权实现级取舍；范围保守）。
- **最终选择**：**① 仅版本对比（viewingHistory）态**。时间线模式 TOC 仍整体隐藏；pending diff 态不带 TOC diff。后续如需再增量。

## D-V1.2-17 · 删除章节是否在 TOC 显示

- **背景**：删除的章节在 target 目录里没有对应条目（无 data-slug 落点），要显示须 base+target 合并、点击跳转 no-op（调研 option G，标注复杂）。但用户图例始终含「🔴删除」。
- **可选项**：① 不显示删除章节（只标存活章节 add/mod，最简）；② 显示删除章节占位（红线、暗色、不可跳转，完整体现 diff）。
- **推荐**：② —— 用户图例含删除、且「体现 diff 差异」自然含删除。
- **谁拍**：lead 代定（贴合用户图例意图）。
- **最终选择**：**② 显示删除章节**：按 base 位置插入一条「红色下划线 + 暗色文本 + 不可点击」的占位条目。

## D-V1.2-18 · 需求B 性能优化范围

- **背景**：lead 用 chrome-devtools 实测 baseline（大文档 1200 行）：**滚动流畅（0 掉帧、0 long task）**；卡顿来源是**渲染瞬间 ~600ms long task**（react-markdown 解析全文）；**实例数不是放大器**（只读 1 实例 593ms ≈ diff 11 实例 596ms，总内容量相同）——调研「N 实例数十倍」热点被实测推翻；滚动既流畅，虚拟化收益存疑且撞 TOC/A3 跳转。
- **可选项**：① 仅 memo 化（低风险纯收益，消除内容不变时的冗余重解析）；② + 按文档长度降级（损形态C体验）；③ + 虚拟化（重、引依赖、撞跳转、数据不支持）。
- **推荐**：① 仅 memo 化。
- **谁拍**：用户。
- **最终选择**：**① 仅 memo 化**（`Markdown` 组件 + components/heading 工厂提模块常量 + React.memo + 段稳定 key）。**不虚拟化、不按长度降级**。

## lead 顺带定（实现级，详见 ADR D-R3-*）

- **章节归属粒度**：子章节(h3)改动**冒泡标记父章节**(h2)（核实指出这是「按下一同级/更高级标题切区间」的必然结果）；空白/格式改动算改动（纯文本 LCS 本性）；标题改名=**拆 del+add 两条**（不做模糊匹配）。
- **真同名同级章节**：text+level 撞 key 时用**标题序列顺序配对**兜底（核实修正②）。
- **不共享 ops**：TOC 归属独立跑一次 lcsDiff + useMemo 缓存，**不拆聚块函数、不碰 D-R7B-04**（核实修正③：性能瓶颈是 react-markdown 非 lcsDiff，无需省这次 lcsDiff）。
