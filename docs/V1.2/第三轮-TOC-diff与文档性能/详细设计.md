## 二、性能 baseline（lead chrome-devtools 实测·大文档 1202 行/16KB）

| 场景 | render long task | 滚动 | 实例数 |
|---|---|---|---|
| 只读全文 | **~593ms**（+99） | avg 16.5ms / **0 掉帧 / 0 longtask** | 1 |
| 版本 diff | **~596ms**（+61） | **0 掉帧** | 11 |

**结论**：① 滚动流畅、卡顿是**渲染瞬间 ~600ms long task**（react-markdown 解析全文）。② **实例数非放大器**（1≈11≈600ms，总内容量相同）——调研「N 实例数十倍」**被实测推翻**。③ lcsDiff 非瓶颈。→ 优化 = **仅 memo 化**（消除内容不变时的冗余 600ms 重解析）；不虚拟化（滚动已流畅、撞跳转）、不按长度降级、不碰 D-R7B-04。

## 三、需求A 设计 · TOC 体现 diff

**算法**（`computeTocDiff(old,new): TocDiffItem[]`，D-R3-03/04，三步 O(行数)）：
1. `parseTocWithLines(content)`：复刻 `toc.ts` 扫描（ATX_HEADING/FENCE/slugify），每标题带 `line`（行下标）。**新函数，不改原 `parseToc`**（守 `toc.test.ts`）。对 base、target 各跑一次。
2. `lcsDiff(splitLines(base),splitLines(target))` 遍历累加 i/j：del→`baseChangedLines[i]=true`、add→`targetChangedLines[j]=true`、equal→两侧未变。
3. 归属：每标题章节区间 = 本标题行 → 下一个**同级或更高级**标题行之前（子章节区间被父区间天然包含 → **子改动冒泡父**，D-R3-03①）。产出 **base/target 合并有序序列**（按位置交织），每项 `diffKind`：target 有 base 无→`add`；base 有 target 无→`del`；两侧都有且区间内有改动行→`mod`；否则 `null`。
   - 对齐 key = **剥离 -1/-2 后缀的 text + level**；**撞 key（真同名同级）→ 按出现顺序配对兜底**（D-R3-04）。
   - 标题改名 → del + add 两条（不模糊匹配）。

**渲染**（`TocSidebar`，D-R3-05）：每目录条按 `diffKind` 加 `borderBottom: 2px solid <KIND_STYLE 色>`（add `#4ade80` / mod `#eab308` / del `#f87171`）+ `paddingBottom`/`marginBottom` 留间距；**无圆点**。`del` 条目：暗色文本（opacity ~0.55）+ **不可点击**（无 data-slug、onClick no-op、cursor default）。`null`（未变）条目渲染同现状（零回归）。

**接线**：viewingHistory 处 `tocDiff = useMemo(() => computeTocDiff(historyBaseContent ?? '', displayContent), [historyBaseContent, displayContent])`（独立 lcsDiff、缓存；**不建 store selector**，D-D3-10）；传 `TocSidebar`。普通查看/pending/时间线态不传 diff（`TocSidebar` 收到无 diffKind → 同现状）。范围仅 viewingHistory（D-V1.2-16）。

## 四、需求B 设计 · 仅 memo 化（D-R3-06）

1. **`Markdown` 组件**（`ArtifactPanel.tsx:53-69`）：6 个 heading 工厂（不依赖 props）+ components 对象 **提到模块级常量**（一次性建）；remarkPlugins 也提常量 `[remarkGfm]`；`Markdown` 包 `React.memo`（props 仅 `children:string`，浅比较命中即跳过重渲染/重解析）。
2. **`InlineDiffView` 段**（:657-669）：equal 段抽成 `React.memo` 子组件（按 `text` 浅比较）；`key` 从 `index` 改**段内容/位置稳定 key**（保 D-R7B-04 段顺序与 block.id 对齐不漂移——稳定 key 须随段内容变而变、不串位）。
3. **`DiffBlockCard`** 包 `React.memo`（按 block.id/state）。
4. ⚠️ AppShell 无 key remount（`AppShell.tsx:1048`）→ 缓存跨切换保留，memo 须自带稳定 key、不靠 remount 清缓存。

## 六、风险护栏（实现期必守）

1. **D-R7B-07 客户端边界**：toc-diff.ts 只值导入 lcs.ts/toc.ts，绝不值导入 pending-change-service/artifact-service（node:fs→全站 500，lint/test/tsc/build 全漏、只 dev+真浏览器暴露）。
2. **D-D3-10 订阅**：tocDiff 用 useMemo 就地算，不建返回新数组/对象的 store selector（否则无限重渲染、只真浏览器暴露）。
3. **D-R7B-04**：本轮不碰聚块对齐；但 T3 段稳定 key 须保段顺序与 block.id 不漂移（改动块增删时段不串位）。
4. **toc.test.ts 契约**：`parseToc` 一字不改，新增 `parseTocWithLines`（否则破 `:28-32/:52` toEqual）。
5. **build 非 oracle**（D-R1-04 Google Fonts）：性能/UI 验收走 chrome-devtools，不认 build/单测证流畅。
6. **memo 正确性**：React.memo 比较函数勿因 props 含函数/新对象而永远不等（heading/components 提常量正是为此）；段 memo 按内容浅比较。
