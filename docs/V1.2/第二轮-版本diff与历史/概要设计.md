# V1.2 第二轮 · 版本 diff 与历史 —— 概要设计

> vibe-coding 第 2 步。模块划分 + 数据流 + 复用清单 + 红线核验。详细到文件/函数级见 `详细设计.md`。
> 承重 file:line 经 lead 亲读复验。

## 一、关键发现（决定本轮架构的事实）

1. **版本是全量快照、永不覆盖**（`artifact-service.ts:37-46,76-79`）：`ArtifactVersion = {version, content(全量), author, note?, createdAt}`，落盘 `managed/<id>/versions/<n>.json`。→ 任意相邻两版的 diff 可**零成本重算**，需求2 无需新存储。
2. **行内 diff 渲染器是纯只读渲染层**（`anchor.ts:24-53` + `ArtifactPanel.tsx:486-657`）：`buildLineDiffSegments(old,new,blocks,changeIdByBlock?)` 按 LCS ops 真实顺序产出 equal/change 段；`DiffBlockCard` 的 ✓/✗ 与状态标全受 `isFullscreen/changeId/resolveBlock` 门控，**不传即纯只读**（`DiffBlocksView` 已是此先例）。→ 需求1「去 ☑️❌」零改 DiffBlockCard。
3. **两版 content 前端已在手**（`useArtifactStore.ts:68,209-220`）：`store.versions[]` 每元素含 content。→ 版本 diff 零额外网络请求。
4. **唯一真缺口**：把两版 content 算成 `DiffBlock[]` 的 `computeReplaceDiffBlocks` 在 server-only 模块（`pending-change-service.ts:129`，引 `node:fs`），客户端**不能值导入**（D-R7B-07 全站 500）。→ 需新建客户端安全聚块函数。
5. **确认后 diff 被删**（`pending-change-service.ts:380` remove）：「逐块确认决策 / agent 身份」重算不回来。→ 本轮只做「内容重算」（D-V1.2-13），不做审计侧车。

## 二、模块划分

| 文件 | 改动 |

| `components/ArtifactPanel.tsx` | ① `viewingHistory` 分支（:387-391）从「只读全文」改为「该版 vs 上一版只读行内 diff」+ 逃生开关（需求1）；② 新增 `[Diff历史]` toggle 按钮（版本下拉右侧）+ 时间线视图（覆盖正文、手风琴就地展开，需求2）；③ `InlineDiffView` 参数化为接受裸 `(old,new,blocks)`（或抽只读兄弟组件）；④ 版本 diff 降级护栏按总块数。 |

## 三、数据流

```
需求1（选历史版即看 diff）:
  版本下拉选 vN → store.selectVersion(N)（已有，拉 historyContent=vN.content）
    + store.versions[] 排序取前驱版 v(N-1).content（D-R2-04）
  → computeVersionDiffBlocks(prev, cur)  [新, 客户端安全]
  → buildLineDiffSegments(prev, cur, blocks)（不传 changeIdByBlock）
  → 复用 InlineDiffView 只读渲染（equal=Markdown / change=DiffBlockCard 无 ✓/✗）
  → 块数 > limit → 降级 DiffBlocksView（并排）

需求2（Diff 历史时间线）:
  [Diff历史] toggle → historyMode=true → 主体区渲时间线
  时间线条目 = store.versions[]（含 note/createdAt/author，已在手）
  点条目 vN → expandedHistoryVersion=N → 懒算该版 vs 前驱 diff
            → 同需求1 渲染管线就地手风琴展开
```

## 四、复用清单（精确到文件:行）

- `lib/domain/lcs.ts`：`lcsDiff/splitLines`（:23-67，客户端安全）——版本 diff 算法基石。
- `lib/artifact-view/anchor.ts`：`buildLineDiffSegments`（:24-53）——传两版 content+blocks 直接复用。
- `components/ArtifactPanel.tsx`：`InlineDiffView`（:486-524 骨架，参数化复用）、`DiffBlockCard`（:536-640，纯渲染 :624-637 无 state 耦合）、`DiffBlocksView`（:646-657 只读先例）、`KIND_STYLE`（:38-45）、`DiffLine`（:659-672）。
- `lib/artifact-view/degrade.ts`：`INLINE_HL_LIMIT=25`（:8）阈值常量。
- `lib/stores/useArtifactStore.ts`：`versions[]`（含 content，:68）、`selectVersion/historyContent`（:222-249）。
- 「照搬蓝本」（**不值导入**）：`pending-change-service.ts:groupOpsToBlocks`（:94-123）——version-diff.ts 照搬其纯算法、只换 id 来源与 state。

## 五、红线核验

- ✅ **不触内容修改红线**：需求1/2 是纯只读重算渲染，不产生 `PendingChange`、不调 `propose_edit`、不调任何写盘/resolve 路径。`DiffBlockCard` 不传 `resolveBlock/changeId/isFullscreen` → `canResolveHere` 恒 false（`ArtifactPanel.tsx:551`），绝不出现可写/可确认元素。
- ✅ **不改 pi 内核**：改动限 `components/` + `lib/artifact-view/` + `lib/stores/`，不碰 `lib/pi/**`、不让内核多发事件。
- ✅ **客户端 bundle 边界**（D-R7B-07）：新算法只从 `lib/domain/lcs.ts` 值导入；`DiffBlock` 仅 `import type`；绝不值导入 `pending-change-service.ts`/`artifact-service.ts`。
- ✅ **无数据库**：零新增存储（重算派），更无 DB。
- ✅ **受管文档去重红线**（第三轮承重墙）：版本快照不泄漏成普通树文件，diff/历史 UI 全挂在 ArtifactPanel 体系内。
