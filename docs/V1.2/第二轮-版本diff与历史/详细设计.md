## 二、需求1 设计 · 版本间行内 diff

**取两版内容**（D-R2-04）：`store.versions[]` 按 version 升序，找选中版 `selectedVersion` 在序列里的**前一个元素**作 base、选中版作 target（target 也可直接用 `historyContent`）。序列首元素（v1）无前驱 → 走「只读全文 + 无对比基准」提示。**不**用「版号-1」（防版本号空洞）。

**算块**（D-R2-01）：`computeVersionDiffBlocks(base, target)`（新建 `lib/artifact-view/version-diff.ts`）→ `DiffBlock[]`。

**渲染**（D-R2-02）：参数化 `InlineDiffView` 让它接受裸 `(oldContent,newContent,diffBlocks)`（pending 调用点 `ArtifactPanel.tsx:497-503` 本就只用到这三样 + changeIdByBlock；版本 diff 不传 changeIdByBlock/resolveBlock/isFullscreen）→ equal=Markdown、change=`DiffBlockCard`（无 ✓/✗ 无状态标）。

**去 ☑️❌**：靠不传 `changeId/resolveBlock/isFullscreen`，`canResolveHere`（:551）恒 false、状态标（:618 `isFullscreen && resolved`）也不显——**零改 DiffBlockCard**（`DiffBlocksView` :646-657 已证此路全屏下也无标）。

**降级护栏**（D-R2-03）：版本 diff 块 state 非 pending → `countPendingBlocks`（`degrade.ts:19`）恒 0。改为按 `blocks.length > effectiveLimit` 判降级到 `DiffBlocksView`（复用 `INLINE_HL_LIMIT`/`FULLSCREEN_INLINE_HL_LIMIT`）。

**逃生口**（D-R2-07/D-V1.2-11）：看历史版 diff 时给一个小开关「对比上一版 ⇄ 只读全文」，默认对比；切只读全文 = 现 :387-391 的渲染。

**入口语义**：改 `viewingHistory` 分支（:387-391）——默认渲版本 diff（除非逃生开关切只读全文、或 v1 无前驱）。

## 三、需求2 设计 · Diff 历史时间线

**入口**（D-R2-06）：工具栏版本下拉右侧加 `[Diff历史]` toggle 按钮（常驻），切 `store.historyMode`。激活高亮；时间线模式下隐藏 `[行内│查看Diff]` 段（:347-366 加 `&& !historyMode` 条件）。

**显示形态甲**（D-V1.2-14）：`historyMode` 时主体区（:384-428 区域）整体渲时间线、覆盖正文。时间线 = `store.versions[]` 倒序（最新在上），每条目显 `v{n} · {note} · {createdAt 相对时间} · {author}`，rollback 条目 note 自带语义。

**手风琴就地展开**（D-R2-05）：点条目 vN → `store.expandedHistoryVersion=N`（标量），**懒算**该版 vs 前驱版 diff（同需求1 管线：取前驱 content → computeVersionDiffBlocks → buildLineDiffSegments → 只读渲染），在条目下方展开；再点收起。v1 条目展开 → 「首版，无对比基准」。

**退出**：再点 `[Diff历史]` 或操作版本下拉 → `historyMode=false` 回正文。

**性能**：只算当前展开的单条（`lcsDiff` O(n*m)），**勿**预算全部版本对。

## 五、风险与护栏（实现期必守）

1. **客户端/服务端边界（最高危·D-R7B-07 已踩过全站 500）**：`version-diff.ts` 只从 `lib/domain/lcs.ts` 值导入、DiffBlock 仅 `import type`，**绝不**值导入 `pending-change-service.ts`。lint/test/tsc/build 全抓不到、**只 dev(Turbopack)+真浏览器暴露**（T1/T2/T3 验收必含 dev 冒烟）。
2. **聚块循环漂移（D-R7B-04）**：`computeVersionDiffBlocks` 聚块循环须与 `anchor.ts:36-49`/`groupOpsToBlocks` 逐字一致，且与 `buildLineDiffSegments` 喂**同一对 (old,new)**，否则 block 与 change 段错位。
3. **zustand 订阅（D-D3-10）**：新增态用标量；任何返回新数组/对象的派生 selector 须 `useShallow`，否则无限重渲染崩、**只真浏览器暴露**（T3 多次点条目专验）。
4. **降级护栏缺失**：版本 diff 块非 pending，旧 `countPendingBlocks` 恒 0，须按总块数另判（D-R2-03）。
5. **去 ☑️❌ 要彻底**：含全屏态——不传 isFullscreen 即可（DiffBlocksView 先例），别误传 panel 的 isFullscreen 给只读版本 diff 的 DiffBlockCard。
6. **build 非有效 oracle**：本环境 `next build` 受 Google Fonts 无外网更早 fail（D-R1-04/D-R7B-07），不作判据；UI 卡铁律走 browser-e2e（`GET /` 200 + pageErrors=0）。
7. **回归面**：改动集中在 `ArtifactPanel.tsx`（已 745 行），须保受管文档 propose→确认链、版本下拉/rollback/删除、TOC/A3、引用到对话框零回归（AC7）。
8. **ultracode 综合不可尽信（D-R1-06）**：实现期任何新增承重论断仍须 lead `file:line` 复核。
