# V1.2 第二轮 · 版本 diff 与历史 —— 需求文档（proposal）

> vibe-coding 第 1 步产物。把「做什么 / 为谁做 / 成功标准」钉死，作为概要 / 详细设计 / 任务划分的唯一约束源。
> 用户拍板见 `../QA/第二轮-版本diff与历史决策.md`（D-V1.2-10~14）；lead ADR 见 `../设计决策记录.md`（D-R2-*）。

---

## 一、项目概述

- **目标**：让受管文档（artifact）的「版本演进」**可视化、可回看**。当前版本管理（D5）只能选历史版看其整篇只读全文，看不出「这一版相对上一版改了什么」，也没有集中回看历次改动的地方。本轮补上这块。
- **性质**：本地单用户工具迭代（V1.2 工作区，承 V1.1 全部已收官能力）。内部工具向。
- **技术栈**（不变）：Next.js 16 App Router + React 19 + TypeScript strict；Zustand；纯文件存储，无数据库 / 鉴权 / 多租户。
- **内核红线**（不变）：不 fork / 不改 pi 内核；artifact **内容修改**必经 `propose_edit` → `PendingChange` → 按块确认 → 才写盘出新版；并发会话 ≤ 3。本轮两功能都是**只读展示/重算**，不引入任何写路径、不碰红线。

## 二、本轮两个功能

### 需求1 · 版本间行内 diff（只读对比）

- file panel 中切换 / 查看不同版本时，能像现有「行内 diff」那样在正文**行内显示版本之间的差异**，让用户看清每个版本「改了哪里、改了什么」。
- **关键区别**：版本间是**已提交的只读历史对比**，**不需要** ☑️❌ 逐块确认按钮（与 pending change 的逐块确认语义不同）。
- **对比基准**（D-V1.2-10）：选中版 vs **紧邻上一版**（vN vs v(N-1)）。v1（首版）无前驱 → 只读全文 + 「首版，无对比基准」提示。
- **入口语义**（D-V1.2-11）：选历史版**即显** diff（推翻祖先 D-D5-4「历史版纯只读」），保留「回到只读全文」逃生开关。
- **视觉**（D-R2-02）：复用第七轮**形态C混合**（equal 段 Markdown 富渲染 + 改动块带边框 git 卡片）。

### 需求2 · Diff 历史（集中回看）

- 新增「Diff 历史」：把历次改动集中列出、方便回看，**易读**、**能行内回看**。
- **落点**（D-V1.2-12）：ArtifactPanel 内新增「**Diff 历史**」时间线 Tab。
- **显示形态**（D-V1.2-14，甲）：点工具栏 `[Diff历史]` 按钮高亮 → 主体区切成时间线**覆盖正文**；时间线纵向列出全部版本（v_n · note · 时间 · 作者），点条目**手风琴就地展开**该版 vs 上一版行内 diff（复用需求1 渲染器）。非浮窗（避开第七轮 A1 全屏浮层冲突）。
- **记录深度**（D-V1.2-13）：**只看每次内容改动**，从版本快照**重算**、**零新增存储**。覆盖 rollback 产生的版本（note 标注语义）。

## 三、成功标准（验收 AC）

- **AC1**（需求1·相邻对比）：建一个有 ≥3 个版本的受管文档，在版本下拉选某历史版 vN → 主体区显示 vN vs v(N-1) 的行内 diff（equal 富渲染 + 改动块带边框卡片）；选 v1 → 只读全文 + 无对比基准提示。
- **AC2**（需求1·只读无确认）：版本 diff 的改动块**没有任何 ✓/✗ 按钮、没有「已确认/已拒绝」状态标**（即使把面板切全屏也没有）。
- **AC3**（需求1·逃生口）：看历史版 diff 时有「回到只读全文」开关，切回后是纯只读全文。
- **AC4**（需求2·时间线）：点 `[Diff历史]` → 主体区切成时间线，列出全部版本（apply pending / rollback 条目可区分 note）；时间线模式下 `[行内│查看Diff]` 段隐藏。
- **AC5**（需求2·行内回看）：点时间线某条目 → 就地手风琴展开该版相对上一版的行内 diff（无 ✓/✗）；点不同条目可多次开合、不崩。
- **AC6**（降级护栏）：版本 diff 改动块数超阈值时自动降级为并排 Diff（不行内崩坏）。
- **AC7**（无回归）：受管文档 propose→按块确认链、版本下拉/rollback/删除、TOC/A3 跳转、引用到对话框等既有行为零回归。
- **AC8**（客户端边界）：dev（Turbopack）下 `GET /` 200、`pageErrors=0`，无 `node:fs` 进客户端 bundle（D-R7B-07 唯一有效 oracle）。

## 四、非目标（本轮不做）

- ❌ 任选两版互比（只做相邻对比；D-V1.2-10）。
- ❌ diff-log 审计侧车（「谁改的 / 确认或拒了哪些块」需新存储；D-V1.2-13）。
- ❌ 跨文档全局 diff 历史面板 / 左栏文件树二级展开（D-V1.2-12 未选）。
- ❌ 非受管文档（dispatch/coding 会话直接写盘的普通文件）的 diff 历史——那走 FileViewer 实时 diff、无版本概念，超本轮范围。
- ❌ 不改 pi 内核、不引入数据库。
