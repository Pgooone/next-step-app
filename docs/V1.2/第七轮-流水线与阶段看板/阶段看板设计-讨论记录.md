## 0. 看板要解决什么

根治 **F14**：sf-mini 13 阶段被迫拆 4 轮派发，**没有任何一个地方能看到"13 阶段整体走到哪了"**，跨轮进度全靠脑记、只能逐轮看派发弹窗。
看板 = 一个视图看全部阶段进度 + 当前在跑第几步 + 每步谁在做 / 产物在哪。

## 2. 卡片样式参考：Kimi agent 集群（已用用户 4 张真实截图核验）

用户需求：很喜欢 **Kimi（Moonshot）agent 集群**内 agent 卡片的样式，希望看板阶段卡做成那种观感。

> ✅ **已核验**：用户提供本目录 4 张真实截图（`*.jpg`），由 4 个 reader 各读一张交叉印证（高一致），下为实证。早先无图的文字调研（A 蜂窝 / B 角色卡之争）**已被截图证伪**——用户喜欢的就是 **B 类「集群内嵌纵向 agent 列表」**，**不是蜂窝**；岔路已定，看板「纵向列表」形态不动。

### 真实卡片设计语言（4 图交叉印证）
1. **分组容器 + 容器头**：顶部一行 `⑂ 分叉图标 + "Agent 集群" + │ + "N 个并行任务"`（N 实时）；下方略不同明度的圆角容器框住该批卡片。⑂ = 「派生出多个」，表达「集群从哪来」。
2. **单卡两行结构**：
   - 行1（身份）：圆头像（拟人插画、各异）+ 拟人名（粗体、卡内最高层级）+ 右上 **两位数零填充序号 `01`**（纯文本灰字、非圆圈徽章）。
   - 行2（任务+状态）：行首 `└` 树形连接符（淡灰、轻父子层级）+ 单行截断任务（中灰 `…`）+ 右侧进度点阵。
3. ⭐ **进度 = 绿色点阵 LED 条**（用户点名「值得学习」）：**3 行 × ~12 列**小方块，从左往右**按列点亮**（左绿=完成、右灰=未完成），整列一起亮、离散刻度；实测 1 列→5 列→全亮；**无百分比 / 进度环 / spinner**。复刻：`grid-cols-12 grid-rows-3` 共 36 个 ~3px `rounded-[1px]` 块、`filledCols = round(progress*12)`、`col < filledCols` 给绿。是区分「子 agent 卡 vs 工具调用卡」的关键（工具卡右侧是 `>` chevron）。
4. **双对齐轴**：头像 / 名 / 任务左对齐，序号 / 点阵右对齐 → 列表整齐。
5. **hover → 右侧 popover**：放大头像 + 露 agent 角色名 + 完整任务 Markdown 富渲染；卡片塞不下的细节移到浮层。
6. **配色 / 质感**：靠明度分层 + 圆角分卡、近乎无边框无阴影、扁平；**唯一强调色 = 绿（仅用在进度点阵）**，其余黑白灰，极克制。⚠️ Kimi 有**深浅两套**：图1/2/4 深色（图2 带蓝霓虹辉光）、**图3 浅色**（近白 `#f6f7f8` 卡片，几乎就是我们 Swiss 浅色想要的）。

### 复刻方向（取神不取形）
**取**：分组头（⑂ + 标题 + N 任务）/ 两行卡 / 右上序号 / `└` 连接线 / **绿点阵进度** / 双对齐轴 / hover 放详情 / 唯一绿强调 + 灰阶。
**不取**：深色 + 蓝霓虹辉光（→ 改 Swiss 浅色细灰描边、去发光、单一克制绿）。

### 映射到「阶段看板」
- 容器头 → 流水线标题 +「N 阶段 · ④/13 运行中」总览。
- 每张阶段卡：行1 = 角色头像 / 图标 + 阶段名（+ 角色名）+ 右上阶段序号；行2 = `└` + 子任务摘要（截断）+ 右侧绿点阵进度。
- 状态：待执行（灰 / 暗置 / 空点阵）· 运行中（点阵部分亮 + 左 3px emerald 条 + 呼吸）· 完成（变灰退场 + ✓ + 满点阵）· 失败（红）。
- hover / 点开 → 详情浮层（角色职责 + 完整子任务 + 产物链接 + 耗时）。

## 4. 视觉定稿（mockup v1→v11，2026-06-27 用户拍板）

**风格：Kimi**（vs Swiss，用户选定）。支持亮 / 暗主题。

- **头像**：dicebear **notionists**（黑白线描人物、每 agent 不同），**本地 `@dicebear/core` 离线生成**（走 npm，不依赖外网）。
- **看板列表**：容器头 `⑂ 流水线名 · ④/13` + 全局进度条 `▰▰▰`；每阶段两行卡（头像 + 阶段名 + 角色 + 右上零填充序号 ／ `└` 任务缩略 + 绿点阵 LED 进度 + 状态徽章 + 行尾 `›`）；完成变灰、待执行暗置、运行态呼吸 + 左 3px 强调条。
- **进度点阵**：3 × 12 绿色 LED 按列点亮。
- **两种交互（分开、勿合并）**：
  - **悬停卡片 → 浮窗预览**（约 330px、可滚动下拉：任务 / 范围 / 上游依赖 / 产物 / 验收清单 / 最近动作 / 耗时）。
  - **左键点击卡片 → 弹出悬浮二级菜单**（非整页跳转）：只读快览该 agent **真实会话内容**（冻结对话 + 产物）+ 底部 agent 切换条；菜单内**「进入完整对话」→ 跳 Next-Step 主对话窗 ChatWindow 续聊**（复用既有 ChatWindow）。
- **配色**：唯一强调色（Kimi 蓝；运行态绿点阵），其余灰阶、克制。

> **定稿已存 git**：`看板视觉定稿-kimi-v11.html`（本目录、独立完整页、双击可开，含 Kimi(A) + Swiss(B) 两块，**选用 Kimi(A)**）。中间稿 v1–v10 仍在（仓库外）`.superpowers/brainstorm/16149-*/content/`（非 git、可回溯重看）。

## 5. 看板行为决策（2026-06-27 用户拍板）

- **运行时交互**：**只读看板（MVP）**——只看进度，不在看板上操作；改产物事后走 ArtifactPanel 按块确认（不破红线、不碰编排状态机）。
- **空态**：**引导建蓝图**——空态卡「+ 新建流水线」直接进蓝图编辑。
- **多 run 历史**：**run 列表可回看**——顶部下拉选历史 run 查看（类似第二轮版本历史；run checkpoint 已落文件、零额外存储）。
- **入口**：**左栏新增「流水线」入口 + 模态**，与 Dispatch 同范式、无新路由。
