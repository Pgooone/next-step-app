## 1. 首用引导 Tour（C2-b 升级 · 本轮 tentpole）

**方案 B 分层**（用户拍板）：默认轻量总览，深度按需。

- **触发与持久化**：首次进工作台（无 `tour-seen` 标记，存 localStorage 或 `~/.pi`）自动启动总览；顶栏常驻 **「🧭 新手引导」** 按钮可随时重看；看完/跳过落 seen 标记不再自动弹。
- **风格**：**暗遮罩 + spotlight 镂空高亮** + 浮动 tooltip 卡（用户拍板，非「仅高亮环」）。tooltip = 层级标签（蓝）+「第 N 步 / M」+ 标题 + 正文 + 步骤点 + 跳过 / 上一步 / 下一步。
- **定位**：spotlight 锚目标元素（`getBoundingClientRect` + 6px padding）；tooltip 优先放右→放不下放左→再不下放下方，最后钳制进可视区（永不溢出/裁切）。
- **技术**（lead ADR · 倾向）：**自定义轻量 overlay**（spotlight 用 `box-shadow: 0 0 0 9999px rgba(...)` 镂空 + tooltip 绝对定位 + 复用项目已有 **gsap** 做过渡），**不引第三方库**（风格可控、零依赖）。每步可带 `before()` 动作：打开真实 Agents/Pipeline 模态、切换模态子视图，再高亮其中真实元素。

> 🔧 **承重订正（2026-06-29 ultracode 对抗审计 → lead file:line 坐实，详 ADR D-R8.5-02）**：深度轨 `before()` 切模态子视图的能力**当前不存在**——`PipelineModal` 的 `tab/view`（:40-41）、`AgentManager` 的 `view`（:93）全是组件内部私有 `useState`，无 prop / forwardRef / imperativeHandle / store / URL 任何外部出口；`AppShell` 只有布尔开关 `agentManagerOpen/pipelineModalOpen`（:60-61），挂载（:1107/:1115）不传初始视图。故深度轨开工前**必须先做组件改造**：给 `PipelineModal` 加 `initialTab`/`initialView`/`initialBlueprintId` props、给 `AgentManager` 加 `initialView`（或把一份 tour 编排 state 上提到 `AppShell`，`before()` set 后再 mount 模态）。这是**附加式加 prop / 提状态、不改既有交互语义**（不是「仅复用现成 open」——原 §1.3/T1 口径已订正）。先走 **mini-spike**（prompt §2）打通「外部设初始 view + spotlight + 空环境」一条最小链路，绿了再铺全步。

## 2. 流水线 UI 修缮（N1 / N2 / N3 / N4）

### 2.1 N1 · 去重两个「新建流水线」
- `PipelineModal.tsx:287` 底部小按钮 label「+ 新建流水线」→ **「✎ 编辑流水线」**（调 `setView('editor')` 不变）；`PipelineBoard.tsx:96` 空态大按钮保持「+ 新建流水线」。语义分开、无需上提 state。

### 2.2 N2 · hover 详情 + click 菜单：边界感知 + 内部滚 + 修闪烁（**不改交互·方案 A**）
现状（`PipelineStageCard.tsx:69-71`，lead 已亲验）：`onMouseEnter`→`StageHoverPreview`（详情速览）；`onClick`→`StageSessionMenu`（二级菜单，内含只读 transcript +「打开受管文档」「进入完整对话」）。**两套独立 state、交互保持不变**。

两个浮层都从「锚卡 `top:calc(100%+6px)`」改为**边界感知 + 内部滚动**：
- **hover 详情**（`StageHoverPreview.tsx`）：锚卡片，**选上下空间更大的一侧**（below/above）展开；`maxHeight = min(330px, 该侧可用空间)` + 内部 `overflow-y:auto`；横向钳制在弹窗内；**不再钳制覆盖卡片**。
- **click 菜单**（`StageSessionMenu.tsx`）：锚**点击点坐标**（类右键菜单），靠底向上翻 / 靠右往左收 + 钳制；菜单 = header + **transcript 区滚动** + **底部「打开文档/进入对话」钉死不跟滚**（flex：header none + transcript flex:1 overflow auto + acts none）。
- **修 hover 闪烁（根因订正 · 2026-06-29 审计 → lead file:line 坐实，详 ADR D-R8.5-04）**：⚠️ 旧表述「详情浮层过高向上钳制盖住卡片→死循环」与**当前代码不符**——两浮层现是**静态 `top:calc(100%+6px)` 下挂**（`StageHoverPreview.tsx:57-58`、`StageSessionMenu.tsx:103-104`）、**已带 `maxHeight+overflowY:auto`**（:67-68）、无 `getBoundingClientRect/scrollTop/clamp/useEffect/onMouseEnter`，所谓「向上钳制」根本不存在。**真根因** = 6px 间隙 + 浮层是 `.brow` 子元素（`PipelineStageCard.tsx:100-102`）但**自身不是 hover 目标**（无 `onMouseEnter`）→ 鼠标穿过 6px 间隙进浮层时离开卡片盒 → `.brow` `onMouseLeave`（:70）→ `setHover(false)` → 浮层消失「够不到」；叠加被 `PipelineModal:116 overflow:hidden` 裁切。**承重修法（非可选加固）**：① **140ms 隐藏防抖** + ② **给浮层加 `onMouseEnter` 取消隐藏**（使浮层可达、可在其中滚动）——这两条才是治本，**别去修不存在的「向上钳制」**。
- **加固**：裁切独立修——定位用 `getBoundingClientRect` 相对弹窗滚动容器换算（含 `scrollTop/Left`）边界感知翻转，或浮层用 `fixed` 定位脱离 `overflow:hidden`。

### 2.3 N4 · 流水线空态加说明
- `PipelineBoard.tsx:80-97` 空态「+ 新建流水线」按钮上方加一两句静态说明：**「流水线 = 多个 Agent 按固定顺序接力、可保存复用；只想临时派一次 → 用上方『快速派发』。」**（与 `DispatchPanel.tsx:285-296` 引导文案同风格）

### 2.4 N3 · run 跑完左栏会话分组自动刷新（补回 · 用户拍板 D-V1.2-65）
> 📌 **补回说明**：N3 在台账 §1 被圈定「全做」，但此前从设计 §2/§7 与任务卡一并掉单（2026-06-29 审计揪出，详 ADR D-R8.5-05）。本节补回，落 T2。
- **缺陷**：run 跑完后左栏会话分组不自动刷新、需手动 Refresh。**透传断点属实**：`PipelineModal.tsx:243-248` 挂载 `PipelineBoard` 时未传 `onSessionsChanged`（该回调 :268 只喂 `DispatchContent`）；`PipelineBoard` props 也无 `onSessionsChanged`。
- **修法**（仿 `DispatchPanel.tsx:117-123` 既有范式）：① `PipelineModal:243` 给 `PipelineBoard` 透传 `onSessionsChanged`；② `PipelineBoard` 加一个「`currentRun` 进终态」effect，用**独立 ref 按 `run.id` 去重**（`currentRun?.id` 在 `PipelineBoard:46` 已可得）触发一次 `onSessionsChanged()`。
- ⚠️ **已接受限制（须写进 AC、防验收假绿）**：轮询/effect 都在 `PipelineBoard` 内，board 在 `view!=='board'` 或模态关闭即卸载（`PipelineModal:155-156` / `AppShell:1115`），轮询随 `:43 clearInterval` 停。故 N3 **只保证「board 开着时 run 跑完自动刷新」**；用户起 run 后关模态再跑完**不刷新**——这与 `DispatchPanel`（:99 同款卸载即停）是**同一既有已接受限制**，本轮**不扩大**修（不上移轮询到 AppShell/store，避免超纲）。验收按此口径判，**不得按「关模态后跑完也必刷」判 PASS**。

## 4. 建项目 C1 · 目录不存在则自动创建（**决策 A · opt-in**）

- **前端**：新建项目表单（`ProjectHome.tsx:137-156` / `ProjectSwitcher.tsx:311-330`）路径框下加 **「□ 目录不存在则自动创建」复选框（默认不勾）**；勾选时请求体带 `createIfMissing: true`，并显示绿字提示「将自动创建该目录后进入项目」。
- **后端**：`project-registry.ts` 的 `validateRoot`（`:133` `statSync` / `:135` 抛 `root 路径不存在`）：当 `statSync` 失败**且** `createIfMissing` 为真时 → `mkdirSync(root, { recursive: true })`（`mkdirSync` 已在 `:4` 导入、`:145` 在用）后**再 `statSync` 复查 `isDirectory()`**（防 TOCTOU）；否则维持原报错。
  - ⚠️ **边界（审计 A2-2）**：`mkdirSync` 自身会对**父级是文件（ENOTDIR）/ 无写权限（EACCES）/ 悬空 symlink（ENOENT）** 抛 fs error，若不接 → 经 `errors.ts:18-20` 映射成 **HTTP 500 且泄 raw 路径/errno**（新手最易踩「路径段是文件 / 写系统目录」）。修：**mkdir 包 `try/catch` → 抛 `ProjectError('INVALID', \`无法创建目录: ${root}（${e.code}）\`)` → 422 友好报错**；补 **ENOTDIR / EACCES 两条边界单测**。
- **改路径**走同一 `validateRoot`（`:109-111`）——create / update 共用此函数，加 `createIfMissing` 分支即同时覆盖（无需单独改 update 路径；审计已证 update 请求体透传 flag 即可）。
- **默认行为不变**：路径已存在直接用；**不勾不动盘**，守「删除项目只移除注册、绝不删盘」的既有谨慎契约。

## 6. 炫酷深色首页（shader 背景 · 试点 · 新增 2026-06-29）

> 设计期用户临时引入「炫酷首页」想法 → 拍板 **「先首页深色试点 · 进 8.5」**。把首页（项目墙）做成 **WebGL shader 流光背景 + 深色玻璃首页**，作为本轮一个**试点**（仅首页、可回退）。

- **范围（试点）**：**仅首页 / 项目墙**改深色 + shader 背景；**工作台保持浅色不动**。配「试点开关」(env/flag) 可一键回退，风险可控。
- **组件**：新增 `components/shader-animation.tsx`（用户提供，three.js 全屏 fragment shader；放扁平 `/components`、非 `/components/ui`，随项目约定）。依赖 `npm i three @types/three`（three ~150KB）。
- **做背景而非全屏替换**：组件由 `h-screen` 改 `position:absolute; inset:0; z-index:0` 垫首页内容后；上叠 `radial-gradient` 暗角 scrim 保文字可读；首页内容（品牌 / 新建项目 / 项目卡）改**深色玻璃**（白字 + `rgba` 半透明 + `backdrop-filter:blur`）。
- **速度**：fragment 时间步进 `+= 0.05` → **`+= 0.018`**（用户拍板，光圈扩散更舒缓；纯参数、可再调）。
- **字体**（深色首页 · `frontend-design` 把关 · ✅ **用户真机定选 B·优雅衬线**）：**标题 Latin `Instrument Serif`（斜体）+ 中文 `Noto Serif SC`**（编辑感/雅致，与炫光背景成「冷峻 × 优雅」对比）；品牌 / UI / 等宽辅以 `Space Grotesk` / `JetBrains Mono`。（备选 A·科技几何 = Space Grotesk + Noto Sans SC 不采用。）⚠️ **构建取不到 Google Fonts（容器代理白名单 = build 已知失败源）→ 真应用必须自托管**：用 `next/font/local` 引自托管 / 子集化 woff2（`Instrument Serif` + `Noto Serif SC` 放 `app/fonts` 或 `public`），**不可** `next/font/google`。
- ⚠️ **承重订正（审计 A3-2 · major）·必做**：T5 把新增 3 字体导向 `next/font/local` 还**不够**——`app/layout.tsx:2` **既有 `import { Noto_Sans_Mono } from "next/font/google"`**（`:7` 实例化、`:24` className，**全仓唯一 google 字体消费点**）不在原 T5 范围、保持不变的话**做完 build 仍 import `next/font/google`、仍按 ADR D-R1-04 失败**。故 **T5 必须顺带把 `layout.tsx` 的 `Noto_Sans_Mono` 也迁成 `next/font/local`**（恢复本机 build 绿），并在 ADR/台账记清「layout 的 mono 也要换」。
- ⚠️ **build oracle 缺口（审计 A3-2）**：本容器 `next build` 因 Google Fonts 不通**跑不到分包阶段**，three.js（WebGL/window 仅客户端）的 SSR / 客户端 bundle 雷（`page.tsx`/`ProjectHome.tsx` 均 `'use client'`，同 D-R7B-07 那类「lint/test/tsc/build 全漏、只 dev + 真浏览器暴露」）**无 build 安全网**。→ T5 验收**强制真浏览器 + dev 控制台 `pageErrors=0` 作为唯一 oracle**，且看 **dev issues overlay + 全量 console**（别只筛 error/warn，沿用 D-R7B-07 / 第二轮教训）；three 组件须 `'use client'` 或 `dynamic(ssr:false)` 隔离。
- **字体获取（开工前 prep · 审计 A3-1）**：仓内**零 woff2 / 零 `app/fonts`**，权威源 Google Fonts 不通 → T5 卡须写**确定性获取步骤**：`--noproxy '*'` 直连 `fonts.gstatic.com` 下载（CLAUDE.md:14 容器可绕代理直连）+ `pyftsubset` 子集化（中文常用字 + 标题用字，Noto Serif SC 全字重过大）+ **开工前先验 woff2 在磁盘且 `next/font/local` 能编译**（编译期从磁盘读、文件缺即 dev 报错），别留到验收。
- **性能 / 无障碍护栏（必做）**：① 离开首页 / 标签隐藏时 **暂停 RAF**（`visibilitychange` + 路由离开 `cancelAnimationFrame`，回来再起，别后台空耗 GPU）；② 尊重 `prefers-reduced-motion` → **静态渐变 / 单帧兜底**；③ `devicePixelRatio` 封顶 2，避免 4K 过度绘制；④ WebGL 不可用时 fallback 纯深色渐变背景。
- **设计稿**：`.superpowers/brainstorm/<session>/content/shader-homepage-demo.html`、`shader-homepage-fonts.html`（原生 WebGL 原样复刻 shader + 深色玻璃首页 + 字体 A/B 切换；不入 git）。
- **触点**：首页（项目墙 `ProjectHome.tsx` / 首页 route）+ 新增 shader 组件 + 字体本地化 + 试点开关。
- **取舍说明**：引入 WebGL + three.js 是**新依赖 + 新视觉方向**；试点先行（仅首页），按反馈再决定是否推广（登录 / 工作台空态等）。
