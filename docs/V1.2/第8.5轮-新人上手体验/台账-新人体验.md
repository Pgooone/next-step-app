## §2.1 首用引导 Tour（初拟 · 待用户确认步骤）

> C2-b 的升级件，本轮 tentpole。第一次进工作台自动弹、可手动重看；高亮目标元素 + 气泡说明 + 上一步/下一步/跳过；看完落标记不再自动弹。

- **触发**：首次进入工作台（`~/.pi` 或 localStorage 无 `tour-seen` 标记）自动启动；顶栏放一个「新手引导」按钮可随时重看。
- **步骤（跟黄金路径走，初拟 5 步，待用户增删）**：
  1. 高亮「新建项目」→「先建个项目；路径填一个文件夹，**不存在可勾选自动创建**」（与 C1 联动）
  2. 高亮左下「Agents」→「在这配协作角色，比如产品经理、架构师」
  3. 高亮左下「Pipeline」→「把任务交给多个 Agent **按顺序接力**＝流水线；只想临时派一次用『快速派发』」（与 C3/N4 联动）
  4. 高亮「▶ 运行」/阶段看板 →「跑起来后这里实时看每阶段进度，点阶段可进会话 / 看产物」
  5. 高亮产物面板 →「产出的文档在这看，带版本和 diff」
- **行为**：上一步 / 下一步 / 跳过 / 完成；遮罩 + spotlight 高亮当前目标；ESC 或跳过即关、落 seen 标记。
- **技术取舍（lead ADR · 待定）**：自定义轻量 overlay（spotlight+tooltip，复用已有 gsap 做动画）**vs** 引入轻量库（driver.js ~5kb）。**倾向自定义**（不加依赖、风格可控、契合既有 Swiss/Kimi 视觉），用 `frontend-design` skill 把视觉与文案。
- **协同**：tour 顺带缓解 C3（高亮入口）、把 C7/C11 的解释也讲到一部分；C3 仍单独做"视觉提权 + 挪 N 浮标"，引导是主线补强、不互斥。
- **成本**：medium（本轮最大一件，需先定步骤 + 一个技术 ADR）；其余 🟢/🟡 多为 trivial/small。
