## 第一轮 · 多 Agent 管理（卡片视觉重构）

- **D-R1-01 · 复用既有纯函数**：头像 / 模型解析复用 `agentColor` / `agentInitial` / `splitModel`（`lib/stores/useAgentStore.ts`），不新造。model 显示用 `splitModel(p.model)?.modelId ?? p.model`（**`?? p.model` 非 `?? '默认'`**，保住无斜杠裸串值）。

- **D-R1-02 · 字段渲染条件**：卡片只读 7 个真实档案字段、零运行态。tools 计数仅 coding 且 >0（doc 模式 tools 被内核忽略、显示会误导）；thinking 仅非 off；role / 计数空则整段隐藏、卡片优雅退化为「头像 + 名 + 模式徽章」。

- **D-R1-03 · 玻璃→Swiss 转向保留 `.glass-card` 类名**（D-V1.2-9 后）：用户拍板转 Swiss 极简，玻璃装饰（极光 / 渐变边 / 品牌环 / 模态环境底）全删，`.glass-card` 基类从「半透明 blur 玻璃」改为「纯色 surface（`var(--bg)` + 细边）」。**保留类名不改**——避免动所有 className 引用；经 grep 实证 `.glass-card` 全仓**仅** AgentManager 卡片 + DispatchPanel 行用，改其样式即全覆盖、无副作用。`.glass-card.agent-card` Swiss 后仅设 `border-radius:10` + hover/focus。

- **D-R1-04 · 不修 build 的 Google Fonts 失败**：`npm run build` 卡在 `app/layout.tsx` 的 `next/font/google` 取 Noto Sans Mono、本环境 `fonts.googleapis.com` 不通（sandbox off 也失败、重试 3 次）。**确认为预存环境限制、非本轮回归**（`layout.tsx` 未碰、失败在字体抓取阶段、早于本轮文件编译）。改 layout 换本地 / 系统字体超本任务范围、**未擅自动**，登记于此。lint + test + 真浏览器三层已验。

- **D-R1-05 · 技术债收敛**：AgentManager 4 处 + DispatchPanel 1 处硬编码「选中态」`rgba(37,99,235,0.x)` → `var(--accent-soft)`（亮暗自适应）；**不动** DispatchPanel `running` 状态徽章色（状态色非选中色、语义不同，误改会让状态徽章跟 accent 漂）。

- **D-R1-06 · 「全局污染真 bug」实为 ultracode 综合夸大**（记教训）：第一轮玻璃设计探索的综合 agent 自称「揪出真 bug——`.glass-card` 全局共享于 file panel/Dispatch/Sidebar、玻璃增强裸挂会污染全站」，并据此定「作用域铁律」。**lead grep 实证该前提为事实夸大**（`.glass-card` 全仓仅 AgentManager 用、无对照物）；另综合给的硬编码 rgba 行号也全错。**教训**：ultracode 综合/reader 会虚构具体事实（路径 / 行号 / 影响面），承重事实须 lead 带 `file:line` 独立复验，勿尽信综合自述。作用域本身无害（未来防护），Swiss 删玻璃后此点 moot。
