## 任务卡与落地

### T1 · 承重墙 spike（lead 复跑通过，探针已删）
faux 起 mode=doc worker（镜像 T2 接线）+ 让一回合 `agent_end.messages` 带 create_artifact toolResult。**实测**：
- 工具集 = `["create_artifact","find","grep","list_artifacts","ls","read"]`（恰 6 项；full 版才含 propose_edit，证剔除生效；profile 带 write/edit/bash 被覆盖）。
- id 抽取：`m.role==='toolResult' && m.toolName==='create_artifact'` → `content.find(c=>c.type==='text').text` → `JSON.parse(text).id`；与 `listArtifacts`/`turn_end.toolResults` 三者**完全一致**。
- 读取激活集 API = 内核 `inner.getActiveToolNames()`；内核事件 API = `inner.subscribe`（dispatch-runner 生产路径走 wrapper `onEvent`）；内核 typed `AgentEndEvent` 无 `willRetry`（运行时 wrapper 透传有，dispatch-runner 不碰 waitForTurnEnd）。
- **结论**：默认 messages 抽取 + listArtifacts 兜底。

### T2 · 接入点 + dispatch 专用 doc 子集（合并原 T3，commit `3d1077b`）
- **doc-tools.ts**：`resolveBackends(deps)` + `makeCreateArtifactTool`/`makeProposeEditTool`/`makeListArtifactsTool`（三段 `defineTool` 逐字搬入、配置不变）；`buildDocTools` = `[create, propose, list]`（签名/返回顺序零改）；新增 `buildDispatchDocTools` = `[create, list]`（无 propose）。
- **doc-session.ts**：`DISPATCH_DOC_SESSION_TOOLS = ['read','grep','find','ls','create_artifact','list_artifacts']` + `assembleDispatchDocSessionOptions(deps)`（用 `buildDispatchDocTools`）。不动 `DOC_SESSION_TOOLS`/`assembleDocSessionOptions`。
- **dispatch-runner.ts**：`runWorker` 增 `projectId`；createAgentSession 前 `docOptions = mode==='coding'?undefined:assembleDispatchDocSessionOptions({projectId,sourceActor:profile.name,cwd}).options` + `codingToolsFallback = mode==='coding'&&tools.length===0?{tools:[...CODING_TOOL_NAMES]}:{}`；`createAgentSession({...options,...(docOptions??{}),...codingToolsFallback,...createOptionsOverride})`。
- **orchestrator.ts**：调用点传 `projectId: task.projectId`。
- **测试**：dispatch-runner.test 4 工具集断言（AC-1/AC-2）；orchestrator.test 经 `freshManagerRunWorker` spread `...args` 自动透传，既有用例仍过。
- 验收：lint 净 / vitest 411（+4）/ tsc 改的 5 文件零新增错误。

### T4 · 产物对账（commit `dc0b4a6`）
- **dispatch-runner.ts**：`WorkerResult` 增 `artifactIds:string[]`（必填、恒返 [])+`createdContent?:string`；新增 `extractCreatedArtifactIds(messages)`（T1 路径）；runWorker 收尾抽 ids + 若非空 `new ArtifactService(new ProjectRegistry()).readCurrentContent(projectId, last)` 回读填 createdContent（**try/catch 兜底**，失败退 undefined）。`extractAssistantText` 零改。
- **orchestrator.ts**：判空 `if(!output.trim() && artifactIds.length===0)` 才 failTask；`assignment.artifactId = artifactIds.at(-1)`；`upstreamOutput = lastArtifactId ? (createdContent ?? output) : output`；writeArtifact 文本保底仍写。
- **dispatch-store.ts**：`Assignment` 增 `artifactId?:string`（无迁移）。
- **测试**：dispatch-runner.test +4 extractCreatedArtifactIds 用例（多 id 按序 / 纯文本 [] / 只认 create_artifact / 非 JSON 跳过）；orchestrator.test 3 桩补 `artifactIds:[]` + 2 对账正反例（AC-3/AC-4/AC-5）。
