## §0 拍板汇总（实现据此，勿再发挥）

| 项 | 定稿 |
|---|---|
| 视觉方向（D-V1.2-2） | 主题自适应玻璃（复用现有 CSS 变量、亮暗双主题、不割裂；**非**深色霓虹照搬） |
| 信息密度（D-V1.2-4） | **6 字段名片式**：名 + 模式徽章 + 模型 + 角色摘要 + 技能/工具/思考计数 |
| 模式信号（D-V1.2-5） | **文字徽章「文档/编码」**（非圆点） |
| 头像（D-V1.2-6） | **加 cyan→violet 品牌渐变光边** |
| 范围（D-V1.2-7） | **含 DispatchPanel 选择卡统一 + 收 4~5 处硬编码 rgba 技术债** |
| lead 默认 | model 放身份行第二行；模式文案短名「文档/编码」；字段渲染条件（tools 仅 coding、thinking 仅非 off、空段隐藏） |

## §0.1 红线铁律（每卡开工前对照，违反即废稿）

1. **纯展示层**：只读 `AgentProfile` 落盘静态字段（name/role/model/mode/skills.length/tools.length/thinkingLevel），**零运行态/零伪造**（无"运行中/Critic/待审"等需会话数据的状态）；模式徽章做**静态文字徽章**，**不用** sf-mini 的呼吸点（呼吸点=在跑语义、用了即造伪状态）。
2. **复用 token**：颜色全走 `var(--*)` + `html.dark` 覆盖；除身份色块（刻意不 token 化的稳定锚点）与 coding 琥珀（语义必需第三色、已给亮暗双前景）外**不引任何新字面业务色**；既有 `.glass-card` 双主题基底**一行不改**，只在 `.agent-card` 上叠伪元素。
3. **不改机制内核**：不碰 `lib/pi/*`、Orchestrator/Artifact/PendingChange、propose_edit 链路；`agentColor`/`agentInitial`/`splitModel`(解析 model) 是既有纯函数，**复用、勿新造**。
4. **⚠️ 玻璃增强必须作用域 `.glass-card.agent-card`，绝不裸 `.glass-card`**——评审实证 `.glass-card` 是全局共享类（file panel / DispatchPanel / SessionSidebar 都用），裸挂会把极光+渐变边+hover 泄漏到全站，是**真 bug**。这是本轮承重铁律。
5. 保留全部既有 `data-testid`（agent-item / agent-name / agent-new-btn / agent-manager / dispatch-agent-item），E2E 不破。

## §2 模块 B · AgentCard（`components/AgentManager.tsx` 现 `:470-527`）—— 批次 2

替换 `AgentCard`。外层 `<button>` **保留** `data-testid="agent-item"` / `data-agent-name={p.name}` / `onClick` / `title`；`className` 由 `"glass-card"` → `"glass-card agent-card"`；`style` 去掉 `aspectRatio:"1"` 改 `minHeight:132`，其余（borderRadius:12, padding:12, flex column, gap:8, overflow:hidden, cursor, font:inherit, textAlign:left）保留。

**三段式结构**：

1. **身份行**（flex, gap:10, alignItems:flex-start, minWidth:0）：
   - 头像 `<div className="agent-avatar" style={{width:40,height:40,fontSize:18,background:agentColor(p.name)}}>{agentInitial(p.name)}</div>`
   - 右列（flex:1, minWidth:0, flex column, gap:2）：
     - 名行（flex, alignItems:center, gap:6, minWidth:0）：名 `<span>`（flex:1, fontSize:13, fontWeight:600, color:var(--text), ellipsis, title={p.name}）+ 模式徽章 `<span className={`agent-badge ${(p.mode??'doc')==='coding'?'agent-badge--coding':'agent-badge--doc'}`} data-testid="agent-card-mode" data-mode={p.mode??'doc'}>{(p.mode??'doc')==='coding'?'编码':'文档'}</span>`
     - 模型行：有 model → `<span className="agent-badge--mono" style={{fontSize:10,color:var(--text-dim),ellipsis}} title={p.model}>{splitModel(p.model)?.modelId ?? p.model}</span>`（**⚠️ 用 `?? p.model` 不用 `?? '默认'`**，保住无斜杠裸串值）；空 → `<span style={{fontSize:11,color:var(--text-dim)}}>默认模型</span>`
2. **role 摘要**（仅 `p.role?.trim()` 非空才渲染）：`<p style={{margin:0,fontSize:11.5,lineHeight:1.5,color:var(--text-muted),display:"-webkit-box",WebkitLineClamp:2,WebkitBoxOrient:"vertical",overflow:"hidden"}}>{p.role}</p>`（空则整段不渲染、不占位）
3. **meta 页脚**（marginTop:"auto", flex wrap, gap:6, alignItems:center）：
   - 技能（仅 `p.skills.length>0`）：`<span className="agent-badge agent-badge--meta" data-testid="agent-card-skills">{p.skills.length} 技能</span>`
   - 工具（仅 `(p.mode??'doc')==='coding' && p.tools.length>0`）：`<span className="agent-badge agent-badge--meta agent-badge--mono" data-testid="agent-card-tools">{p.tools.length} 工具</span>`
   - 思考（仅 `p.thinkingLevel && p.thinkingLevel!=='off'`）：`<span className="agent-badge agent-badge--meta" data-testid="agent-card-thinking">{{low:'低',medium:'中',high:'高'}[p.thinkingLevel]} 思考</span>`
   - 三者全不满足 → 页脚不渲染，卡片优雅退化为「头像+名+模式徽章」

**头像**：维持 `agentColor`（8 色板哈希、同名恒同色、刻意不 token 化作为身份锚点）+ `agentInitial`（首字符大写、空名→「?」、白字）；唯一新增 = `.agent-avatar::after` 品牌环（模块 A 已定义）；位置从独占一行挪到身份行左侧。

**`splitModel` 来源**：复用现有 model 解析 helper（B2 注入按首个 `/` 解析、UI 模型下拉应已有）。**团队 grep 确认其真实路径与签名再用，勿新造**；若不存在则退化为直接显示 `p.model`。

**新增 data-testid**（供 E2E 断言）：`agent-card-mode` / `agent-card-skills` / `agent-card-tools` / `agent-card-thinking`。

- **技术债收敛**：把硬编码"选中态底色" `rgba(37,99,235,0.x)` 统一为 `var(--accent-soft)`（lead grep 实测，2026-06-22 订正综合给的错误行号）——

## §5 验收要点（真浏览器，lead 独立复验 + team e2e）

1. **⚠️ 全局污染回归（最关键、承重铁律）**：确定性验证极光/渐变边/hover 上浮**只作用 `.agent-card`**——file panel 的文件项、DispatchPanel 其它 `.glass-card`、SessionSidebar 等**无极光、无渐变描边泄漏、无上浮**。（判据：截图对照 + 可在 console 查非 agent-card 的 glass-card 计算样式无 `::before` 极光。）
2. **亮/暗双主题**：6 字段齐显；模式徽章 doc=中性 / coding=琥珀（暗色 amber 对比达标）；头像品牌环可见；hover 上浮+描边转 accent；focus 环。两主题各截一图。
3. **字段条件**：doc agent **不显**「工具」计数；thinking=off **不显**思考；role 空的 agent 卡片优雅退化（不留空白段）；model 裸串（无 `/`）正确显示原值不被吞成「默认」。
4. **不回归**：点卡进二级菜单（起会话/配置/删除）、新建空卡、CRUD、hover；`pageErrors=0`。
5. **DispatchPanel**：选 agent 行新玻璃风 + 模式徽章；选中态底色亮暗自适应（`--accent-soft`）；勾选/MIN-MAX 逻辑不破。
6. **门禁**：`npm run lint && npm run test && npm run build` 全绿。
7. **基线/前后对比**：team e2e 出重构前后对比图（用户要的"当前卡片基线截图"并入此处），lead 亲 Read 核对。

## §6 风险（来自评审，team 实现/验收时盯）

- **等高参差**：`minHeight:132` 内容驱动 + `line-clamp:2` 封顶 + 页脚 `marginTop:auto` 缓解，但做不到像素级等高（同行卡底边略参差）——可接受。
- **coding 琥珀第三色**：全卡唯一主题外语义色，已给亮暗双前景；token 洁癖可接受。
- **`mask-composite:exclude`**（渐变边 + 头像环）：极老浏览器降级为无环/无高光边，但仍是干净玻璃卡——Next 16 锁现代浏览器，可接受。
- **`-webkit-box`+`WebkitLineClamp`**：事实标准、主流支持，须真浏览器验渲染一致。
- **token 多**：`globals.css` diff 偏大（一次性设计系统投入，后续看板/流水线可复用同套色）。
- **网格列宽 150→176**：同弹窗宽每行卡片略减、agent 多时纵向更长——密度取舍。
