# 第二轮 · 视觉精修（用户反馈驱动）

> 第一轮卡片重构（批次1+2）真浏览器验收功能全 PASS 后，用户看截图反馈视觉问题 + 选 B（强化玻璃）。本轮 = 缺陷驱动直做（不另出三件套，但记 QA + 本 spec + 双层验收 + 红线核对）。
> 关联：`详细设计.md`（第一轮规格、红线 §0.1 仍生效）、`../QA/第一轮-多Agent管理决策.md`（D-V1.2-8）。

## 诊断（lead 亲看 `diag-agents.png` / `diag-dispatch.png`）

1. **字号散乱**：name 13 / model 10 / role 11.5 / chip 10 / 默认模型 11 —— 不成阶梯、11.5 是脏值。
2. **卡片高度参差**：role 用 2 行 `-webkit-line-clamp`，内容多寡致卡片忽高忽低（架构师 2 行角色 + 3 chip 很满；Critic 很空），网格看着乱。
3. **玻璃显 flat**：玻璃卡浮在**纯白/纯深的模态底**上，玻璃「没有背景可 frost」→ 质感出不来。**这是「玻璃弱」的真因**——光提极光不够，得给模态一层环境底让玻璃有东西可透。
4. **DispatchPanel 选择行**：仍是旧 flat 灰行（`var(--bg-hover)` + 勾选框 + 名 + 角色），无头像/模式/模型，与新卡片两套风格 → 「非常乱」。

## 精修方案

### A · 统一字号阶梯（消散乱，全整数）
| 层级 | px / weight | 用于 |
|---|---|---|
| 头像首字母 | 16 / 700 | avatar |
| 主标题 | 13 / 600 | 卡名 |
| 次要 | 12 / 400 | role |
| 三级 | 11 | 模型 / 模式徽章 / 计数 chip / 「默认模型」 |

去掉 11.5；「默认模型」与 model 同为 11。

### B · AgentManager 卡片等高 + 干净布局（`components/AgentManager.tsx`）
- **role 改 1 行 ellipsis**（去 `display:-webkit-box`/`WebkitLineClamp`，改 `whiteSpace:nowrap` + `overflow:hidden` + `textOverflow:ellipsis`）→ 所有卡同结构、**等高**，消除参差。
- 字号按 A 阶梯统一（name 13 / role 12 / model·badge·chip 11）。
- card `padding:13`；身份列 `gap:2`、card `gap:8` 保持统一；`minHeight:132` 不变。

### C · 玻璃强化（B，让玻璃真"读"得出）
- **模态加环境底**（核心）：AgentManager 模态面板 +（顺带）DispatchPanel 模态面板，加一层**极淡 cyan/violet 径向渐变底**（新 token `--modal-glow-1/2`，亮 ~4-6% / 暗 ~8-10%），让玻璃卡有东西可 frost、卡片"浮"出来。
- 卡极光提透明度：`--card-glow-1/2` 亮 `0.07/0.06 → 0.11/0.09`、暗 `0.12/0.12 → 0.17/0.15`。
- `--glass-edge` 略提明度（亮 `0.10 → 0.13`；暗 `0.18 → 0.22`）。
- hover（上浮 + accent 描边 + 极光增强）保持。

### D · DispatchPanel 选择行玻璃化（`components/DispatchPanel.tsx`，= 批次3）
把 flat 灰行升级为**横向玻璃卡行**，与 AgentManager 同视觉语言：
- 布局：左 勾选框（保留）+ 小头像（`.agent-avatar` 缩到 32，agentColor + initial + 品牌环）+ 文字列 [名 + `.agent-badge` 模式徽章] / [模型 splitModel · role 1 行]。
- 行用 `.glass-card`（**轻量、不加 `.agent-card` 极光**，避免长列表太花；加 hover 提亮即可）。
- 选中态：`border-color: var(--accent)` + `background: var(--accent-soft)`（替换硬编码 `rgba(37,99,235,0.06)`）。
- **保留** `dispatch-agent-item` testid + 勾选 / MIN-MAX 逻辑。
- 复用纯函数 agentColor/agentInitial/splitModel。

### E · 技术债收敛（同详细设计 §4，行号就地核准）
AgentManager 4 处选中芯片底 `rgba(37,99,235,0.10)` + DispatchPanel 选中行 `0.06` → `var(--accent-soft)`；**不动** DispatchPanel `running` 状态徽章色。

## 红线（仍守，逐条）
纯展示层（只读档案字段、零运行态）· 复用 token 不硬编码 · 不改机制/内核 · 玻璃增强仍作用域 `.agent-card`（模态环境底是模态面板自身样式、非全局 `.glass-card`）· 保留全部 testid。

## 验收（双层）
- 逻辑层：lint + test 门禁绿（build 受 Google Fonts 环境限制、非本轮回归，沿用第一轮结论）。
- 真浏览器（lead 亲看截图）：① 字号统一无散乱；② 卡片等高、网格整齐；③ 玻璃明显"浮"得出（模态环境底 + 卡极光）；④ DispatchPanel 行新玻璃风、与卡片一致；⑤ 亮/暗双主题；⑥ pageErrors=0 + 不回归（点卡进菜单/勾选/MIN-MAX）。
