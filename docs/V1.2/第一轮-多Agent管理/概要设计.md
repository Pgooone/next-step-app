# V1.2 第一轮 · 多 Agent 管理 —— 概要设计（high-level / 模块划分）

> vibe-coding 第 2 步。输入 `需求文档.md`，本文只做一件事：**划分模块**（职责单一、可独立理解/测试/修改）。
> 本增量（增量一）= agent 管理卡片视觉重构（主题自适应玻璃），**纯展示层**。视觉细节规格见 `详细设计.md`（由 ultracode 设计探索工作流产出）。

## 模块职责表

| 模块 | 落点文件 | 一句话职责 | 独立性 |
|---|---|---|---|
| **玻璃设计 token** | `app/globals.css`（`:21-53` 变量 + `:270` 区 `.glass-card`） | 升级 `.glass-card` 与玻璃相关 CSS 变量（模糊层次 / 半透明 / 渐变描边 / hover），亮暗双主题；**纯全局 CSS** | 可独立改、独立看（任何用 `.glass-card` 处即时生效） |
| **AgentCard（卡片展示）** | `components/AgentManager.tsx`（`AgentCard` 子组件，约 `:470`） | 卡片结构与内容：头像 + 名称 + 角色 + 模型 + 模式徽章 + 技能/工具计数；用 token、不硬编码颜色 | 依赖玻璃 token；可独立验证（渲染/点击/亮暗） |
| **AgentList（网格）** | 同文件 `AgentList`（约 `:404`） | 网格列宽/间距适配新卡片尺寸；「新建 Agent」空卡 + 空状态文案同步新风格 | 依赖 AgentCard |
| **DispatchPanel 选择卡 + 技术债**（D-V1.2-7 确认在内） | `components/DispatchPanel.tsx`（`:304-380`）+ AgentManager 4 处 | agent 选择行升级玻璃风 + 复用模式徽章；收敛 4~5 处硬编码 `rgba(37,99,235,0.x)` → `--accent-soft` | 独立（依赖批次1 token） |

## 依赖与批次（实现顺序）

- **批次 1：玻璃 token**（`globals.css`）—— 无依赖、是卡片视觉的基座。
- **批次 2：AgentCard + AgentList**（`AgentManager.tsx`）—— 依赖批次 1 的 token。
- **批次 3：DispatchPanel 统一 + 技术债收敛**（D-V1.2-7 用户拍板纳入）—— 依赖批次 1 token；批次 2 收口后再开。

> 同一文件内 AgentCard 与 AgentList 紧邻、一并改即可视作**一张卡**（批次 2）。批次 1 与批次 2 跨文件、可分别 commit（[[next-step-commit-per-task]] 细粒度回退）。

## 模块独立性（vibe-coding 原则 2）

- 玻璃 token 在 `globals.css`、卡片结构在 `AgentManager.tsx`，**两文件互不重叠**——可分别理解、分别在真浏览器验证、分别 commit。
- 卡片**只读** `AgentProfile` 真实字段（`name / role / model / skills / tools / mode / thinkingLevel`），**无运行态数据来源**（卡片是「档案」非「在跑会话」）。

## 红线对照（北极星不变量）

纯展示层改动：**不改** `agent-profile-store` / `profile-session-wiring` / 提议工具 / dispatch / pi 内核；不引入数据库/重型依赖；并发会话 ≤3 不涉及。✓ 每张卡开工前再对照一次。

## 与详细设计的衔接

`详细设计.md` 给出每个模块的**具体规格**（`.glass-card` 升级 CSS、AgentCard 字段与布局、头像处置、hover、亮暗适配），来源 = ultracode「主题自适应玻璃」设计探索工作流（研读 sf-mini 参考+现状 → 3 角度方案 → 对抗评审综合）。其中**留给用户拍板**的取舍（展示字段 / 头像形态 / 是否本增量统一 DispatchPanel）走 `QA/` 记录后再定稿详细设计。
