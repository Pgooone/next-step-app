## 目标
FileExplorer 顶部加「受管文档」分组（点开进 ArtifactPanel）；普通树按**绝对路径**去重剔除物化的 crd.md。

## AC
- [ ] FileExplorer 新增 3 prop：`projectId` / `projectRoot` / `onOpenArtifact(id)`。
- [ ] 受管文档可折叠分组子组件：`useEffect([projectId, refreshKey])` 自取 `GET /api/projects/[id]/artifacts`；projectId 为空跳过不发请求/不渲染该组；加载中/空数组不渲染该组（树保持干净）；复刻 ArtifactPicker.tsx:101-127 的 `title + kind·v{currentVersion}` 排版改树行样式；点击 → `onOpenArtifact(a.id)`。
- [ ] **绝对路径去重**（D-V3-04，⚠️ 非裸名）：`managedAbsPaths = new Set(artifacts.filter(a=>a.filePath).map(a=>normalize(joinFilePath(projectRoot, a.filePath))))`；普通树 `roots.map`(:259) 前 `.filter(node => !managedAbsPaths.has(node.fullPath))`。旧 artifact 的 `undefined filePath` 经 `.filter(Boolean)` 跳过。
- [ ] **首帧 gate**：普通树列表渲染 gate 在 `artifacts!==null`，消除 crd.md 首帧闪现。
- [ ] 复用 `lib/file-paths.ts` 的 `joinFilePath` / 归一函数（与 FileExplorer.tsx:39 `fullPath` 同口径，确切函数名以该文件实际导出为准），**不手拼字符串**。
- [ ] 逻辑层单测：managedAbsPaths 过滤后根级不含受管绝对路径节点；`filePath=undefined` 不污染 Set；`cwd≠projectRoot` 时不误剔同名节点（绝对路径不等）；子目录 artifact 的绝对路径能命中剔除。
- [ ] 门禁绿 + 更新 `components/README.md`（受管分组并入 FileExplorer）。

## 复用
ArtifactPicker 行渲染（title + kind·v）；`listArtifacts` API；`lib/file-paths.ts`。
**关键**：去重 key **必须绝对路径**——cwd≠projectRoot 真实可能（handleCwdChange AppShell.tsx:175 切 cwd 不切项目、cwd 下拉列全局会话目录），裸名会误剔/张冠李戴/子目录漏剔（对抗 blocker）。受管机制层不动。
