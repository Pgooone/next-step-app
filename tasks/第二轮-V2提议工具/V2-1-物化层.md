# V2-1 · 文档物化层（地基）

> 批次 1，无依赖。详见 `../../docs/第二轮-V2提议工具/详细设计.md` · V2-1。

## 目标
扩展 `lib/domain/artifact-service.ts`：把受管 artifact 的"当前版"物化成项目里**真实 `.md` 文件**。

## AC
- [ ] `Artifact` 类型增 `filePath?: string`（物化真实文件相对 projectRoot 的路径），写进 `artifact.json`。
- [ ] `createArtifact`：生成 `filePath = sanitizeFileName(title) + ".md"`（🔧 取舍2：物化到项目根）；建 v1 后物化 content 到 `<projectRoot>/<filePath>`。
- [ ] `submitVersion`：提交新版后物化新 content 到真实文件（覆盖 = 当前版）。
- [ ] `rollback`：回退后物化回退版 content 到真实文件。
- [ ] **外部编辑保护（D-V2-06）**：materialize 覆盖真实文件前，读真实文件 vs 即将被覆盖的 version 快照比对，不一致抛 `EXTERNAL_MODIFIED`(→409) 拒绝物化（防外部手改被静默覆盖丢失；首版 create 不比对）。
- [ ] `sanitizeFileName` **复用 `lib/domain/orchestrator.ts` 既有实现**（已覆盖空/纯点/分隔符/控制字符/兜底 + 5 单测）、提共享位置；补边界：清洗后为空/纯点/超长(length 上限)/与磁盘同名 .md 避让。
- [ ] 物化用原子写（临时 + rename，仿既有 `atomicWrite`）。
- [ ] 单测：建/改/退后真实文件内容正确；artifact.json 含 filePath；非法/空/纯点/超长 title 安全清洗；同 title 第二次建不覆盖；**外部改真实文件后 submitVersion 抛 EXTERNAL_MODIFIED**。
- [ ] 门禁绿（test + tsc + lint）+ 更新 `lib/domain/README.md`。

## 复用
既有 `createArtifact / submitVersion / rollback / atomicWrite`；只在末尾追加物化，不动既有版本/侧车逻辑。
