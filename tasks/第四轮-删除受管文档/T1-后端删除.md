# T1 · 后端删除（service.deleteArtifact + DELETE 路由 + 单测）

> 批次 1，无依赖（先做）。详见 `../../docs/第四轮-删除受管文档/详细设计.md` §四 + ADR D-V4-01~04。

## 目标
机制层补「删」：`ArtifactService.deleteArtifact` 彻底删除（侧车目录 + 物化 .md），`DELETE /api/artifacts/[id]` 暴露，带 If-Match 乐观锁。与 `createArtifact` 对称的结构操作，**不走 propose**（D-V4-02）。

## AC
- [ ] `lib/domain/artifact-service.ts` 新增 public `deleteArtifact(projectId, id, input?: { ifMatch?: number }): void`：`readMeta`(NOT_FOUND) → `assertVersionMatch`(409) → 容错删物化 .md（`materializedPath` 缺/无 filePath/被外删 → `rmSync(force:true)` 静默跳过，D-V4-03）→ `rmSync(artifactDir, {recursive,force})` 清 meta+versions+pending。复用现成私有方法、不改其它方法。
- [ ] `app/api/artifacts/[id]/route.ts` 在现有 GET 旁新增 `DELETE` handler（仿 rollback 路由：`parseIfMatch` → `findArtifact` 反解 projectId → `deleteArtifact` → 204；错误经 `domainErrorResponse`）。
- [ ] 单测 `lib/domain/artifact-service.test.ts` 增：删后 `listArtifacts` 不含、侧车目录消失（`existsSync` false）、物化 .md 消失、带 pending 的删后 pending 目录清；不存在 id→NOT_FOUND；If-Match 不符→VERSION_CONFLICT；无 filePath 的旧 artifact→不抛只删侧车；.md 已被外删→不抛正常删侧车。
- [ ] 门禁绿（lint + test + tsc）。

## 注意
- `deleteArtifact` 返回 void（或删前的 Artifact 元数据，按需）；路由返回 204 No Content。
- 既有三写方法（create/submit/rollback）/ 侧车格式 / 提议工具**一行不动**——只新增。
- `rmSync` 若 artifact-service 未 import 需补 import（`fs`）。

## 复用
`readMeta` / `assertVersionMatch` / `materializedPath` / `artifactDir`（均现成私有）；rollback 路由（`parseIfMatch` / `findArtifact` / `domainErrorResponse` 范式）；artifact-service.test.ts 既有夹具。
