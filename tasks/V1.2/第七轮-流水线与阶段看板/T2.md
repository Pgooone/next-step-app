# T2 · 数据层：pipeline-store + pipeline-run-store + 蓝图 API + 校验

## 目标
落两个领域 store（蓝图 CRUD + run checkpoint 读写）+ 蓝图 CRUD 的 API 路由 + 保存校验，全部原子写文件、无 DB、hermetic 单测（仿 dispatch-store.test）。

- [ ] 定义类型 `PipelineBlueprint` / `PipelineStageSpec`（§2.1）：`id`/`projectId`/`name`/`stages`/`createdAt`/`updatedAt`；stage = `order`/`agentId`/`subTaskTemplate`
- [ ] 定义类型 `PipelineRun` / `PipelineRunStage`（§2.2）：含 `status`(复用 `DispatchStatus`)/`currentStageIndex`/`cancelRequested`/`failedReason`/`stages`；stage 含 `statusDetail?:"queued"`/`sessionId`/`artifactId`/`startedAt`/`finishedAt`
- [ ] 新增 `lib/domain/pipeline-store.ts`：class PipelineStore，方法 `create`/`get`/`list`/`update`/`delete`/`write`（签名见详细设计 §3.1）
  - [ ] 原子写**逐字照搬** `dispatch-store.ts:134-138`（tmp = filePath+".tmp-"+process.pid → writeFileSync → renameSync）
  - [ ] 目录定位 `factoryDir(projectId) = join(registry.get(projectId).root, ".pi", "factory", "pipelines")`（仿 `dispatch-store.ts:56-58`）
  - [ ] `findBlueprint(pipelineId)` 跨项目兜底（仿 `dispatch-store.ts:111-117`，cancel/run 备用；store 主用 `get(projectId, pipelineId)`）
