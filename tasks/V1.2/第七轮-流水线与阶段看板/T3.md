## 验收标准（AC）
- AC-2：每阶段跑完即销毁会话释放并发槽，串行运行中任一时刻活会话恒 ≤1（不碰 bySession/removeOwner）。
- AC-3：≥4 阶段一键起 run 连续跑完、不被旧 10min idle 占槽拖住。
- AC-4：多 run 并发受并发上限限流、自然轮转排队、不互相饿死/不死锁；不依赖单飞守卫即成立。
- AC-5：并发槽占满时下一阶段排队等待、不被 60s 超时误杀；看板显「排队中·等会话槽」（看板渲染在 T4/T6，本卡保证 statusDetail="queued" 落盘）。
- AC-6：N≥4 下游阶段首条消息含全部已完成上游产物摘要（非只紧邻一跳）。
- AC-7：dev 重启后 running 且当前阶段 sessionId 不在活 registry → GET run 读时对账判 failed。
