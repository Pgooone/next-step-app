# T1 · 承重墙 spike（A 冻结释槽 + B re-attach 续聊）

> 承重：是 ｜ 依赖：—（开工第一步）｜ 对应 AC：AC-2 / AC-3（T1-A）· AC-12（T1-B）命门 ｜ 权威：详细设计 §四承重墙 + §五任务表 T1

## 目标
开工前两条承重命门各跑一个 hermetic spike 单测、lead 独立复跑：T1-A 证「每阶段 evict 后并发槽确定性回落、≥4 阶段不卡 10min idle」，T1-B 证「被 evict 销毁过的 dispatch worker 会话能 re-attach 装回受限 doc 工具集（含 propose_edit）而非 generic」。**T1 不绿不开 T3（编排器）/ T6（进会话）。** 探针收官删。

---

## T1-A · 冻结释槽真能腾出并发槽（F16，AC-2/AC-3 命门）

### 证什么
`runPipeline` 每阶段跑完调 `evictAgentSessions` 后，活会话 registry 计数**确定性回落**——≥4 阶段连跑，任一时刻活 worker 会话 ≤1，无须等 `rpc-manager.ts:52` 的 10min idle。

### 子任务（最小可执行 · checklist）
- [ ] 写 spike：注入 faux registry（桩 `globalThis.__piSessions` 或经 evict 的 `EvictDeps` / concurrency-gate 的 `activeCount` DI）
- [ ] 写 faux `runWorker`：注册一个假 inner 进 registry、返回 `{reason:'completed', artifactIds:['a'], ...}`，**留在 registry 模拟 completed 不自销毁**（坐实现状痛点）
- [ ] 模拟 ≥4 阶段连跑：每阶段 evict 后断言 `activeSessionCount()` 归零
- [ ] 断言第 i+1 阶段 `acquireSlot({timeoutMs:Infinity})` 立即放行（不卡 idle）
- [ ] 收官删探针

### 坐实（已 Read 实证）
现 orchestrator 全文无 `destroy()`/`evictAgentSessions`：`orchestrator.ts` 0 处、completed 路径 `dispatch-runner.ts:161-163` 不 abort 不 destroy → 新增 evict 后释槽生效。

### 验法 / 不绿则降级
- 验法：注入 faux `globalThis.__piSessions` + faux runWorker，断言计数轨迹（每阶段 evict 后归零、下阶段 acquireSlot 立即放行）。
- 不绿则降级：T1-A 是冻结模型命门，不绿则整轮停工、回头改设计（无降级路径）。

## T1-B · dispatch worker 会话能 re-attach 装回 doc 工具（AC-12 命门，D-V1.2-42）

### 证什么
一个由 `runWorker` 起 + `setOwner` 写了归属、**且被 evict 销毁过**的 dispatch worker 会话，经 `resolveOrReattachSession`（`session-reattach.ts:97`）能被 `lookupProfile` 反查命中、走 `reattachProfileSession` 重建**受限 doc 工具集**（含 propose_edit、人在场合法）而非 generic（反塞 write/edit/bash）。

### 子任务（最小可执行 · checklist）
- [ ] 写 hermetic spike：经 `resolveOrReattachSession` 的 DI 缝（`getOwner`/`lookupProfile`/`reattach`/`startGeneric`，`session-reattach.ts:75-87`）
- [ ] 真实 evict 后断言：会话被销毁（registry 无）后，给定 bySession 仍有归属（evict 不碰 owner map，`evict-agent-sessions.ts:11-13` 实证）
- [ ] 断言 resolve 走 reattach 分支、装回受限集（**复用 `profile-session-wiring.test.ts:576` 既有 reattach 工具集断言**）
- [ ] 收官删探针

### 已有强证据（已 Read 实证、降低风险但不免 spike）
`设计决策记录.md` **D-R5-06** 已文档化「dispatch worker 会话进 bySession 后，idle re-attach 续聊得交互式 `DOC_SESSION_TOOLS`（7 项含 propose_edit、`doc-session.ts:28`；区别于 headless 的 6 项 `DISPATCH_DOC_SESSION_TOOLS`、`doc-session.ts:44`——re-attach 是人在场的交互式 doc 会话、合法）」。本轮差异 = 销毁途径从「idle 自动」变「主动 evict」，但二者都只 destroy wrapper、不删 jsonl（`evict-agent-sessions.ts:14-16` 实证），故 `resolveSessionPath`（`app/api/agent/[id]/route.ts:24`）仍能定位 jsonl、`resolveOrReattachSession` 仍走 `getOwner`→命中分支（`session-reattach.ts:115-125`）。

### 验法 / 不绿则降级
- 验法：hermetic，经 DI 缝 + 真实 evict 后断言走 reattach 分支、装回受限集。
- **不绿则降级**：T1-B 不绿 →「进入完整对话」降级为「只读快览」（不阻塞其余 MVP，但 **AC-12 改判「降级达成」须用户确认**）。
