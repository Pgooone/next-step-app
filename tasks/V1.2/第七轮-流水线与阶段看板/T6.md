## 已知限制（T6 暴露的 pre-existing 缺陷，决策 D-R7-07 决策2、留二期）
- **跨 run evict-by-agentId 误杀复活会话**：`orchestrator:187 doEvict(projectRoot, stage.agentId)` 按 **agentId** 逐出该 agent 名下全部存活会话（`sessionsForAgent`），非只逐本阶段 sessionId。用户经「进入完整对话」复活的某 agent 会话，会被**后续另一条复用同 agent 的 run** 跑完阶段时连根 destroy。T6 是首个让 worker 会话在 run 结束后复活的功能、从而暴露此 pre-existing 缺陷（单 run 测不出）。**T6 不改 evict**（改逐出粒度=动 owner map 语义、撞第五轮红线）；单 agent 单 run 的 AC-12 不受影响、正常 PASS。**若要修**：需扩 owner map 语义区分「worker 临时会话 vs 用户接管的交互会话」，是独立一轮的工作量。已向用户披露。
