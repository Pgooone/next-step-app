# T1 · 承重墙 spike：in-flight 计量「绝不漏算」命门

> 承重：是（全盘成败所系）｜ 依赖：— ｜ 对应 AC：AC-6 ｜ 权威：详细设计 §二承重墙

## 目标
活进程打点坐实「计量改 in-flight 后，真正在跑的会话绝不被漏算」——这是方案 B 唯一致命路径（`pipeline-orchestrator.ts:127` timeoutMs:Infinity 排队，漏算→超额放行且无超时兜底）。**未 GO 不开 T2。**

## 子任务（最小可执行）
- [ ] lead 先带 file:line 复核内核 `isStreaming` 时序（`agent.js` run 起置 true / finally `finishRun` 置 false；`d.ts:266` getter）——承重事实不照搬调查、亲核。
- [ ] 探针/驱动（本环境 `~/.pi/agent` 配 deepseek、可真起会话，参 `scripts/verify-t6-drive.mjs` 法）：在会话 onEvent 回调 + 轮询点打 `inner.isStreaming` 时间戳。
- [ ] **验①全生命周期**：worker 单回合（createAgentSession→registerInnerSession→send→`agent_end && !willRetry`）内 `isStreaming` 全程 ===true，覆盖 LLM 流 + 工具执行 + 多轮 follow-up。
- [ ] **验②半拍边界**：agent_end 监听回调内读 ===true、下一 microtask 后 ===false（坐实「多算半拍」是保守不漏放）。
- [ ] **验③非 completed 路径**（必须覆盖）：timeout 阶段 + worker 抛错阶段 → abort 后 `isStreaming===false`、不计 in-flight（证 F16 在非 completed 路径也不复发）。
- [ ] **验④N 阶段串行**：`inFlightSessionCount` 每阶段末（completed 留 Map）正确回落到「下阶段可放行」、peak≤1。
- [ ] **负对照**：构造「假在跑」会话（streaming 永不翻 false）→ 证漏算则下阶段死等超时被测出。
- [ ] **变异检查**：`inFlightSessionCount` 误写数 `size`（不过滤）→ N 阶段累积断言必红。

## 验收标准（go/no-go）
- ①②③④全绿 + 负对照 + 变异检查能抓回归 = **GO**（开 T2）。
- 任一打点显示 `isStreaming` 在回合内出现 false 窗口（漏算可能）= **NO-GO** → 退 evict-by-sessionId、回报用户重新拍板。

## 门禁 / 纪律
- lead 独立复跑探针（不认 impl 自跑）；faux Map「三处闭包同一对象 + 负对照 + 变异检查」纪律（沿 T1/T3 第七轮）。
- 探针文件待 T3 收官删（命门事实迁详细设计 + ADR D-R8-*）。
