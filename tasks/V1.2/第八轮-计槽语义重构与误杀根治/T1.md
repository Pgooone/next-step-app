# T1 · 承重墙 spike：by-sessionId 逐出粒度正确（hermetic）

## 目标
hermetic 单测坐实 `evictSession(root, sessionId)` **只 destroy 目标 sid、不误删同 agent 其他活会话、流式先 abort、不碰 owner-map**。这是 evict-by-sessionId 的命门（纯应用层、零内核时序依赖，**比方案 B 的全局计量时序命门轻得多、无需活进程打点**）。**未 GO 不开 T2。**

## 子任务（最小可执行）
- [ ] lead 先带 file:line 复核 `evict-agent-sessions.ts:41-48` 现有「isStreaming→abort→destroy」配方 + `:11-13` owner-map 红线注释。
- [ ] 探针（hermetic，无需起真会话）：faux `fauxMap`（唯一计数源）+ `ownerBySession` 反查；放 2 个会话 sidA/sidB **同属 agentX**（模拟「本阶段 worker sidA + 用户接管 sidB」）。
- [ ] **断言①（误杀根除直接证据）**：`evictSession(root, sidA)` 后 `fauxMap` 只剩 sidB（sidA.destroy 被调、真删 Map）、`sidB.isAlive()` 仍 true。
- [ ] **断言②（流式守卫）**：sidA.inner.isStreaming=true → 先 `send({type:'abort'})` 再 destroy（spy 顺序）。
- [ ] **断言③（owner-map 红线）**：evict 后 `getOwner(cwd, sidA)` 仍返 agentX（只 destroy registry、绝不碰 bySession/removeOwner）。
- [ ] **负对照**：`destroyDeletesMap=false`（destroy 只翻 alive 不删 Map）→ `fauxMap.size` 不回落，证「size 回落」非 vacuous。
- [ ] **变异检查**：`evictSession` 误写成「逐出该 agent 全部会话」（退化回 by-agentId）→ 断言① 必红（sidB 被误删），证测试能抓住误杀回归。
