## 四、调查挖出的真实漏洞（后续 roadmap，按优先级；全部细节在 `docs/流程漏洞审查.md`）

> 方法：agent team 4 路并行 + lead 亲自核实了 3 个关键漏洞（不只信 agent）。

### P0 · 承重墙断裂【已亲验｜v2 命门】
**产物按块确认闭环在运行时是断的**：
- **§B**：拦截器 `assembleArtifactGuardOptions`（`lib/pi/artifact-guard.ts:149`）**全仓库零运行时调用者**——起会话链路从没装它。
- **§A**：dispatch 产物用 `writeArtifact` 落普通 `.md`（`lib/domain/orchestrator.ts`，D-C-1），不进受管系统。
- 后果：北极星红线①「artifact 改动必经按块确认」**生产未强制**；HITL UI（D3/D4/D5）只能显示手动/测试造的 pending，真实 Agent 写产物不触发。
- **v2 头号修法**：起 profile 会话时调 `assembleArtifactGuardOptions({sourceActor,cwd,...})` 把 `options` 并入 `createAgentSession`（用法注释见 `artifact-guard.ts:143-147`）；dispatch 产物改走 `createArtifact` 受管落盘。

### P1 · 该补（低成本高收益）
- **§I 新手第一道坎（一行可修）**：`ProjectHome.handleEnter`(69-72) 进项目**不设 cwd**，而 `ProjectSwitcher.handleSelect`(66-72) 设了 → 经项目墙进**全新项目**时 cwd 空 → New/Skills 禁用、**主对话标题不渲染、新手看不到主对话入口**。修法：`handleEnter` 比照补 `onProjectSelected(p.root)`。⚠️ **M7 E2E 当年用「种子会话锁 cwd」规避了它、验收全绿没暴露——真实新手撞得上**。
- **补全局 toast 系统**：当前整个应用**没有 toast**——@转交/起会话/确认/回滚成功都无回执、@转交失败只 `console.error`（`AppShell:307`，这是 M8 自己的）。补一个能一次解决「操作无回执」一大片。
- §C 引用注入（`editTarget` 划选引用不进 `handleSend`）、§F dispatch 刷新即丢（无持久化/无列表端点）、§H 首回合流式 token 可能丢。

### P2 · UX 文案（纯文案、零风险）
空状态零引导（产物/Agent/会话）、`Get Started` 文案过时（`AppShell:888` 仍是 pi-web 旧文案）、`@` placeholder 无提示、高危操作（全部✓/回滚）无二次确认。

> 关键认知：**绝大多数"漏洞"不是 bug，是"分期未接线"或"可补文案"。真承重墙只有 P0 一条。**
